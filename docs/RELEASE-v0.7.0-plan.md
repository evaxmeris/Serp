## 🎯 版本概述

### 版本目标

Sprint 7 的核心目标是 **完成认证系统重构**，解决原 NextAuth Beta 版本与 Next.js 16 的兼容性问题，采用简化的 JWT 认证方案，确保系统可用。

### 背景

- 原方案使用 NextAuth v5 Beta 版本
- NextAuth v5 Beta 与 Next.js 16 存在兼容性问题（CSRF 验证、Cookies API 不兼容）
- 需要快速修复，确保系统能够按时交付

## 📦 v0.7.0 变更内容

### 新增功能

1. **全新简化 JWT 认证系统**
   - 基于 `jose` 库的 JWT 生成和验证
   - HttpOnly Cookie 存储 token
   - 7 天过期时间
   - 完全兼容 Next.js 16

2. **登录速率限制**
   - 按 IP 地址限制
   - 默认 15 分钟内最多 5 次失败
   - 自动清理过期记录
   - 返回 429 状态码和重试时间

3. **完整的认证 API**
   - `POST /api/auth/login` - 用户登录
   - `GET /api/auth/me` - 获取当前用户
   - `POST /api/auth/logout` - 用户登出

### 架构变更

| 项目 | 旧方案 | 新方案 |
|------|--------|--------|
| 认证库 | NextAuth v5 Beta | `jose` + 自定义实现 |
| 会话管理 | NextAuth 内置 | JWT + HttpOnly Cookie |
| 兼容性 | ❌ 与 Next.js 16 不兼容 | ✅ 完全兼容 |
| 可控性 | 🟡 黑盒 | ✅ 完全可控 |
| 依赖大小 | 重量级 | 轻量级 (jose 仅 8KB) |
