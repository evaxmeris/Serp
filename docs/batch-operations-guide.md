## 概述

### 什么是批量操作？

批量操作允许你一次性对多条数据进行相同操作，大大提高工作效率。

### 支持批量操作的模块：

| 模块 | 导入 | 导出 | 删除 | 其他操作 |
|------|:----:|:----:|:----:|:--------:|
| 产品管理 | ✅ | ✅ | ✅ | - |
| 订单管理 | - | - | - | 确认/发货 |
| 客户管理 | ✅ | ✅ | - | 打标签 |
| 供应商管理 | ✅ | ✅ | - | 分类调整 |
| 库存管理 | - | - | - | 入库/出库/盘点 |

## 通用操作流程

### 批量导入流程

```
准备 CSV 文件
  ↓
选择导入文件
  ↓
系统自动预览
  ↓
确认字段映射
  ↓
开始导入
  ↓
等待导入完成
  ↓
查看导入结果
  ↓
下载错误报告（如果有错误
```

### CSV 文件格式要求

1. **编码：** UTF-8
2. **分隔符：** 逗号 (,)
3. **第一行：** 必须是表头
4. **文件大小：** 建议不超过 10MB
5. **单文件条数：** 产品最多 1000 条，其他模块最多 500 条

### 通用响应格式

所有批量操作都返回统一格式：

```json
{
  "success": true,
  "message": "操作完成",
  "results": {
    "total": 100,
    "success": 95,
    "failed": 5,
    "errors": [
      {
        "row": 10,
        "message": "SKU 重复"
      }
    ]
  }
}
```

## 产品管理批量操作

### 批量导入

**功能：** 批量导入产品数据

**CSV 表头示例：**

```csv
SKU,名称,分类ID,采购价,销售价,最小库存,最大库存,供应商ID
SKU001,夜光万圣节笔,1,5.00,15.00,100,1000,SUP001
SKU002,化妆刷套装,2,10.00,29.00,50,500,SUP002
```

**必填字段：** `SKU`, `名称`

**可选字段：** `分类ID`, `采购价`, `销售价`, `最小库存`, `最大库存`, `供应商ID`

**特殊说明：**
- SKU 必须唯一，重复会被拒绝
- 价格使用小数格式
- 如果分类ID不存在会自动创建分类

**API：** `POST /api/products/batch-import`

---

### 批量导出

**功能：** 导出产品列表为 CSV

**筛选选项：
- 按分类筛选
- 按供应商筛选
- 按库存范围筛选

**输出包含所有产品信息：SKU、名称、分类、价格、库存等

**API：** `GET /api/products/batch-export`

---

### 批量删除

**功能：** 批量删除产品

**安全机制：
- 会检查关联数据（订单、库存）
- 如果有关联数据会提示你确认
- 删除会记录操作日志

**API：** `DELETE /api/products/batch-delete`

---

## 订单管理批量操作

### 批量确认订单

**功能：** 一次性确认多个订单

**前置条件：** 订单状态必须是 "pending"

**操作结果：** 所有选中订单状态改为 "confirmed"

**API：** `POST /api/orders/batch-confirm`

---

### 批量发货

**功能：** 批量对多个订单执行发货操作

**操作结果：
- 订单状态改为 "shipped"
- 自动创建对应的出库单
- 自动扣减库存

**需要提供：** 每个订单物流单号（可选）

**API：** `POST /api/orders/batch-ship`

---

## 客户管理批量操作

### 批量导入

**功能：** 批量导入客户数据

**CSV 表头示例：**

```csv
名称,邮箱,电话,分类,标签
张三,zsan@example.com,13800138000,批发客户,新客户,
李四,li@example.com,13900139000,零售客户,老客户,VIP
```

**必填字段：** `名称`, `邮箱`

**特殊说明：** 邮箱必须唯一，重复会被拒绝

**API：** `POST /api/customers/batch-import`

---

### 批量导出

**功能：** 导出客户列表为 CSV

**输出包含：** 所有客户信息，包括分类、标签、创建时间等

**API：** `GET /api/customers/batch-export`

---

### 批量打标签

**三种模式：

| 模式 | 说明 |
|------|------|
| `add` | 添加标签（保留原有标签） |
| `remove` | 移除标签 |
| `replace` | 替换所有标签 |

**请求示例：**

```json
{
  "ids": ["id1", "id2", "id3"],
  "mode": "add",
  "tags": ["VIP", "重点客户"]
}
```

**API：** `POST /api/customers/batch-tag`

## 库存管理批量操作

### 批量入库

**功能：** 一次性处理多个商品入库

**请求示例：

```json
{
  "items": [
    {
      "productId": "sku1",
      "quantity": 100,
      "costPrice": 5.00,
      "purchaseOrderId": "po123"
    },
    {
      "productId": "sku2",
      "quantity": 50,
      "costPrice": 10.00,
      "purchaseOrderId": "po123"
    }
  ]
}
```

**操作结果：** 增加对应商品库存，创建入库单记录

**API：** `POST /api/inventory/batch-receive`

---

### 批量出库

**功能：** 一次性处理多个商品出库

**操作结果：** 减少对应商品库存，创建出库单记录

**API：** `POST /api/inventory/batch-issue`

---

### 批量盘点

**功能：** 批量更新库存盘点结果

**说明：** 直接将库存数量调整为盘点数量

**API：** `POST /api/inventory/batch-count`

## 常见问题 FAQ

### Q: 批量导入支持哪些文件格式？

**A:** 目前支持 CSV 格式，CSV 可以从 Excel 导出，导出时选择 CSV(逗号分隔)格式即可。

---

### Q: 一次最多可以导入多少条数据？

**A:** 产品最多 1000 条，其他模块最多 500 条。如果数据更多，建议分成多个文件分批导入。

---

### Q: 导入失败了怎么办？

**A:** 系统会生成详细的错误报告，告诉你哪一行有什么错误。修正后修正 CSV 文件重新导入即可。

---

### Q: 批量删除可以撤销吗？

**A:** 当前版本不支持撤销删除撤销操作，操作前请确认选中数据。重要操作前建议先导出备份。

---

### Q: 为什么有些操作按钮是灰色的？

**A:** 可能的原因：
1. 你没有选择任何数据
2. 你的权限不足（需要管理员或经理权限
3. 数据状态不允许该操作（例如已发货订单不能重复确认

---

### Q: 导入时提示 "SKU 重复" 怎么办？

**A:** SKU 必须唯一。请检查 CSV 中是否有重复，或者数据库中已存在该 SKU。如果需要更新，请先删除原数据后重新导入。

---

### Q: 导入时提示 "邮箱重复" 怎么办？

**A:** 每个客户邮箱必须唯一，请检查 CSV 或者是否已有客户使用该邮箱。

---

### Q: 批量操作会记录日志吗？

**A:** 是的，所有批量操作都会记录操作人、操作时间、操作内容，方便后续审计。

---

### Q: 批量打标签三种模式有什么区别？

**A:**
- `add`：在原有标签基础上添加新标签，不删除原有标签
- `remove`：从客户身上移除指定标签，保留其他标签
- `replace`：删除所有原有标签，替换为新标签

请根据需要选择合适的模式。

---

### Q: 批量导入失败会回滚吗？

**A:** 系统会逐行处理，成功的会保存，失败的会跳过，不会回滚已经成功的数据。这样你只需要修正失败的行重新导入即可。

---

### Q: 有没有示例 CSV 文件模板可以下载？

**A:** 系统在导入页面提供下载，你可以下载模板填写数据后导入。
