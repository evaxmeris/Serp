# Sprint 8 - 跨境电商多平台支持

**版本目标：** v0.8.0  
**Sprint 周期：** 2026-03-25 ~ 2026-04-07（2 周）  
**优先级：** P0 - 核心功能

---

## 一、Sprint 目标

实现跨境电商多平台基础支持，包括：
1. 用户角色权限系统
2. 多平台订单接入（阿里国际站优先）
3. 多仓库管理
4. 多平台产品刊登基础

---

## 二、用户故事

### US-001：用户角色权限管理
**作为** 创始人  
**我希望** 为团队成员分配不同角色和权限  
**以便** 确保数据安全和职责分离

**验收标准：**
- [ ] 支持 5 种预定义角色（创始人/运营/业务员/美工/采购）
- [ ] 支持自定义角色和权限
- [ ] 数据访问隔离（业务员只能看自己的客户）
- [ ] 操作日志记录

### US-002：阿里国际站订单同步
**作为** 业务员  
**我希望** 自动同步阿里国际站订单到 ERP  
**以便** 统一处理订单，减少手工录入

**验收标准：**
- [ ] 支持阿里国际站账号授权
- [ ] 自动拉取新订单（每 15 分钟）
- [ ] 订单状态同步（待付款/待发货/已发货/已完成）
- [ ] 支持手动触发同步

### US-003：多仓库管理
**作为** 运营  
**我希望** 管理多个仓库（本地仓/海外仓）  
**以便** 优化库存分布和发货效率

**验收标准：**
- [ ] 支持创建多个仓库
- [ ] 仓库类型标识（本地/海外/第三方）
- [ ] 库存按仓库隔离
- [ ] 支持仓库间调拨

### US-004：多平台产品刊登
**作为** 运营  
**我希望** 将产品一键发布到多个平台  
**以便** 提高上架效率

**验收标准：**
- [ ] 产品库支持多平台关联
- [ ] 支持选择目标平台
- [ ] 自动映射产品属性
- [ ] 刊登状态跟踪

## 三、任务分解

### 3.1 数据库变更
- [ ] 新增 `user_roles` 表
- [ ] 新增 `permissions` 表
- [ ] 新增 `platform_accounts` 表
- [ ] 新增 `platform_orders` 表
- [ ] 修改 `users` 表（增加 role_id）
- [ ] 修改 `warehouses` 表（增加 type 字段）
- [ ] 修改 `products` 表（增加 platform_ids 字段）

### 3.2 后端 API
- [ ] POST /api/roles - 创建角色
- [ ] GET /api/roles - 获取角色列表
- [ ] PUT /api/roles/:id/permissions - 设置权限
- [ ] POST /api/platform-accounts - 绑定平台账号
- [ ] GET /api/platform-orders/sync - 同步订单
- [ ] POST /api/products/:id/publish - 发布产品到平台

### 3.3 前端页面
- [ ] /settings/roles - 角色管理页面
- [ ] /settings/platforms - 平台账号管理
- [ ] /orders/platform - 平台订单列表
- [ ] /products/publish - 产品刊登页面

### 3.4 集成任务
- [ ] 阿里国际站 API 对接
- [ ] 订单同步定时任务
- [ ] 平台 Webhook 处理

## 四、技术设计

### 4.1 权限模型（RBAC）
```
User → Role → Permissions
```

### 4.2 订单同步架构
```
定时任务 → 平台 API → 订单队列 → 订单处理 → ERP 订单
```

### 4.3 数据隔离策略
- 行级权限（业务员只能访问自己的数据）
- 角色权限中间件
