**文档版本：** v1.0  
**创建日期：** 2026 年 3 月 13 日  
**优先级：** P0（业务刚需）  
**状态：** 待评审

## 1. 引言

### 1.1 目的
本文档定义 Trade ERP 系统中产品调研模块（Product Research）的详细功能需求、业务规则和技术约束，为开发团队提供清晰的实现指南。

### 1.2 业务背景
在海外仓备货决策流程中，采购团队需要对潜在产品进行系统性调研和评估。当前缺乏统一的产品信息管理工具，导致：
- 产品信息分散在 Excel、聊天记录中
- 无法横向对比多个候选产品
- 缺乏标准化的评估维度
- 调研结论无法有效传递给备货计划

### 1.3 适用范围
- **目标用户：** 采购经理、采购专员、运营经理
- **适用场景：** 美妆、个护、人体护理电器等品类的选品调研
- **业务流程：** 备货计划 → 产品调研 → 调研结论 → 备货决策

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 产品调研 | 对单一产品的系统性信息收集和评估过程 |
| 动态属性 | 根据产品品类自动加载的可配置属性字段 |
| 调研结论 | 对产品的最终评估结果（推荐/备选/淘汰） |
| 产品对比 | 同时对比多个产品的关键指标和属性 |

```
        ↓                                                        ↓
┌─────────────────────────────────────────────────────────────────┐
│                      关联备货计划                                │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 功能需求

### 3.1 品类与属性模板管理 (P0)

#### 3.1.1 品类管理
- **FR-001:** 系统应支持创建、编辑、删除产品品类
- **FR-002:** 品类应支持层级结构（一级品类 → 二级品类）
- **FR-003:** 品类应包含：品类名称、品类编码、描述、状态（启用/禁用）

#### 3.1.2 属性模板管理
- **FR-004:** 每个品类可配置专属的属性模板
- **FR-005:** 属性类型支持：文本、数字、日期、下拉选择、多选、布尔值
- **FR-006:** 属性可配置：字段名、字段类型、是否必填、默认值、选项列表（下拉/多选）
- **FR-007:** 属性模板应支持版本管理（修改后不影响历史调研数据）

### 3.2 产品调研录入 (P0)

#### 3.2.1 创建调研
- **FR-008:** 用户可创建新产品调研，选择产品品类
- **FR-009:** 选择品类后自动加载对应的属性模板
- **FR-010:** 调研初始状态为"草稿"

#### 3.2.2 基本信息录入
- **FR-011:** 必填字段：产品名称、品牌、平台（Amazon/TikTok 等）、采购价、售价
- **FR-012:** 选填字段：产品链接、供应商信息、备注
- **FR-013:** 支持上传产品图片（最多 9 张，单张≤5MB）

#### 3.2.3 动态属性录入
- **FR-014:** 根据品类模板动态渲染属性输入表单
- **FR-015:** 必填属性未填写时禁止提交
- **FR-016:** 支持保存草稿，随时续填

#### 3.2.4 调研状态管理
- **FR-017:** 状态流转：草稿 → 调研中 → 待审核 → 已完成
- **FR-018:** 状态变更需记录操作人和时间

### 3.3 产品对比功能 (P0 - 核心)

#### 3.3.1 对比列表管理
- **FR-019:** 用户可选择多个产品（2-10 个）加入对比
- **FR-020:** 对比列表应支持保存和命名
- **FR-021:** 支持从对比列表中移除产品

#### 3.3.2 对比视图
- **FR-022:** 表格形式展示对比产品，每列一个产品，每行一个属性
- **FR-023:** 自动识别并高亮显示关键差异（价格差异>20%、评分差异>1 星等）
- **FR-024:** 支持自定义显示/隐藏对比字段
- **FR-025:** 支持按任意字段排序

#### 3.3.3 对比维度
- **FR-026:** 基础信息对比：名称、品牌、价格、平台
- **FR-027:** 动态属性对比：根据共同品类属性自动对齐
- **FR-028:** 调研结论对比：推荐等级、评估分数

### 3.4 调研结论 (P0)

#### 3.4.1 结论类型
- **FR-029:** 结论类型：推荐（优先备货）、备选（可考虑）、淘汰（不推荐）
- **FR-030:** 每种结论类型需填写理由说明

#### 3.4.2 评估打分
- **FR-031:** 支持多维度打分（1-5 分）：市场需求、利润空间、竞争程度、供应链稳定性
- **FR-032:** 自动计算综合得分（可配置权重）

#### 3.4.3 结论审核
- **FR-033:** 采购专员提交的结论需采购经理审核
- **FR-034:** 审核意见：通过、驳回（需填写原因）、修改后通过

### 3.5 备货计划关联 (P1)

#### 3.5.1 关联关系
- **FR-035:** 调研结论为"推荐"的产品可关联到备货计划
- **FR-036:** 一个产品可关联多个备货计划（不同批次）
- **FR-037:** 关联时自动同步产品基本信息和属性

#### 3.5.2 数据追溯
- **FR-038:** 备货计划可查看关联的产品调研详情
- **FR-039:** 产品调研页面显示关联的备货计划列表

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 | 测量方法 |
|------|------|----------|
| 页面加载时间 | ≤2 秒 | 从点击到页面完全渲染 |
| 对比功能响应 | ≤3 秒 | 10 个产品对比数据加载 |
| 图片上传 | ≤5 秒/张 | 5MB 图片上传完成 |
| 并发用户 | ≥50 人 | 同时在线操作 |

### 4.2 可用性要求
- **系统可用性：** ≥99%（工作时间 9:00-21:00）
- **数据持久性：** 自动保存草稿，防止数据丢失
- **操作可撤销：** 关键操作（删除、状态变更）支持撤销

### 4.3 安全性要求
- **权限控制：** 基于角色的访问控制（RBAC）
- **数据隔离：** 不同公司/团队数据隔离
- **操作审计：** 关键操作记录日志（创建、修改、删除、审核）

### 4.4 兼容性要求
- **浏览器：** Chrome 90+、Safari 14+、Edge 90+
- **移动端：** 支持 iPad 访问（只读模式）

## 5. 数据模型

### 5.1 核心实体关系

```
┌──────────────┐       ┌──────────────┐       ┌──────────────┐
│   品类表     │ 1:N   │  属性模板表  │ 1:N   │  产品调研表  │
│  category    │───────│  template    │───────│   research   │
└──────────────┘       └──────────────┘       └──────────────┘
                                                       │
                                                       │ N:M
                                                       ↓
┌──────────────┐       ┌──────────────┐       ┌──────────────┐
│  备货计划表  │←──────│  调研结论表  │ 1:N   │  对比列表表  │
│   plan       │       │  conclusion  │───────│  comparison  │
└──────────────┘       └──────────────┘       └──────────────┘
```

### 5.2 主要数据表

#### 5.2.1 产品调研表 (research)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | ✓ | 主键 |
| name | VARCHAR(200) | ✓ | 产品名称 |
| category_id | UUID | ✓ | 关联品类 |
| brand | VARCHAR(100) | ✓ | 品牌 |
| platform | VARCHAR(50) | ✓ | 销售平台 |
| purchase_price | DECIMAL(10,2) | ✓ | 采购价 |
| sale_price | DECIMAL(10,2) | ✓ | 售价 |
| status | VARCHAR(20) | ✓ | 草稿/调研中/待审核/已完成 |
| conclusion | VARCHAR(20) | | 推荐/备选/淘汰 |
| created_by | UUID | ✓ | 创建人 |
| created_at | TIMESTAMP | ✓ | 创建时间 |
| updated_at | TIMESTAMP | ✓ | 更新时间 |

#### 5.2.2 属性模板表 (template)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | ✓ | 主键 |
| category_id | UUID | ✓ | 关联品类 |
| field_name | VARCHAR(100) | ✓ | 字段名 |
| field_type | VARCHAR(20) | ✓ | text/number/date/select/multi/boolean |
| is_required | BOOLEAN | ✓ | 是否必填 |
| default_value | VARCHAR(500) | | 默认值 |
| options | JSON | | 选项列表（select/multi） |
| version | INT | ✓ | 版本号 |

#### 5.2.3 产品属性值表 (research_attribute)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | ✓ | 主键 |
| research_id | UUID | ✓ | 关联调研 |
| template_id | UUID | ✓ | 关联模板 |
| field_value | TEXT | | 属性值 |

## 6. 接口设计

### 6.1 REST API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/categories | 创建品类 |
| GET | /api/categories | 获取品类列表 |
| PUT | /api/categories/:id | 更新品类 |
| DELETE | /api/categories/:id | 删除品类 |
| POST | /api/research | 创建调研 |
| GET | /api/research | 获取调研列表 |
| GET | /api/research/:id | 获取调研详情 |
| PUT | /api/research/:id | 更新调研 |
| POST | /api/research/:id/images | 上传图片 |
| POST | /api/comparisons | 创建对比列表 |
| GET | /api/comparisons/:id | 获取对比详情 |
| PUT | /api/research/:id/conclusion | 提交调研结论 |
| POST | /api/research/:id/conclusion/review | 审核结论 |

### 6.2 关键接口示例

#### 创建产品调研
```json
POST /api/research
{
  "name": "某品牌美容仪",
  "categoryId": "uuid-here",
  "brand": "BrandName",
  "platform": "Amazon",
  "purchasePrice": 150.00,
  "salePrice": 299.00,
  "attributes": [
    {"templateId": "uuid-1", "value": "红色"},
    {"templateId": "uuid-2", "value": "2000mAh"}
  ]
}
```

#### 产品对比
```json
POST /api/comparisons
{
  "name": "美容仪对比-Q1",
  "researchIds": ["uuid-1", "uuid-2", "uuid-3"]
}
```
