### AC-001: 用户登录成功

**前置条件**: 用户已经注册，邮箱 `test@example.com`，密码 `Password123!`

**测试步骤**:
1. 发送 POST 请求到 `/api/auth/login`，body `{ "email": "test@example.com", "password": "Password123!" }`
2. 检查响应
3. 检查响应头中的 Set-Cookie

**预期结果**:
- ✅ 响应状态码: `200 OK`
- ✅ 响应 JSON: `{ "success": true, "user": { "id": "...", "email": "test@example.com", "name": "...", "role": "..." } }`
- ✅ Set-Cookie 包含 `auth-token=...`
- ✅ Cookie 属性: `HttpOnly`
- ✅ Cookie 属性: `SameSite=lax`
- ✅ Cookie 属性: `max-age=604800` (7天)
- ✅ 生产环境: `Secure` 属性存在

### AC-008: 用户登出

1. 发送 POST 请求到 `/api/auth/logout`，携带 `auth-token` Cookie

3. 再次调用 `/api/auth/me`

## 安全验收

| 检查项 | 验收标准 | 状态 |
|--------|----------|------|
| JWT 签名密钥 | 必须从环境变量读取，不硬编码 | ✅ 已满足 |
| 密钥强度 | NEXTAUTH_SECRET 长度至少 32 字符 | ⚠️ 需部署时检查 |
| HttpOnly Cookie | auth-token 设置 httpOnly=true | ✅ 已满足 |
| SameSite Cookie | SameSite=lax 降低 CSRF 风险 | ✅ 已满足 |
| 生产环境 Secure | 生产环境自动设置 Secure=true | ✅ 已满足 |
| JWT 过期时间 | 过期时间为 7 天，不设置永不过期 | ✅ 已满足 |
| 速率限制 | 登录接口限制 5 次/15分钟 | ✅ 已满足 |
| 密码加密 | 所有密码使用 bcrypt 加盐哈希 | ✅ 已满足 |
| 环境变量检查 | 启动时强制检查 NEXTAUTH_SECRET，未设置抛出错误 | ✅ 已满足 |

**安全验收结论**: 所有强制要求已满足，等待部署时检查密钥强度。
