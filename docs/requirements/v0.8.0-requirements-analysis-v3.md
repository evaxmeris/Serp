## 1. 项目概述

### 1.1 项目背景

**团队构成：** 14 人
- 创始人 × 1
- 阿里运营 × 1
- 业务员 × 3
- 美工 × 2
- 采购 × 1
- 出纳 × 1
- 财务 × 1
- 文员 × 1
- 亚马逊负责人 × 1
- 亚马逊运营 × 1

**业务版图：**
- 当前：阿里国际站（已运营）
- 规划：亚马逊、TikTok、独立站

**品类：** 美妆、个护、成人用品、个人护理电器

**仓储模式：** 多平台海外仓备货

### 1.2 核心目标

1. 实现全团队协作数字化
2. 精准利润核算到每个订单/每个业务员
3. 自动化提成计算与绩效考核
4. 规范化多级审批流程
5. 支持多平台业务扩展与数据隔离

## 2. 角色需求分析（10 种角色）

### 2.1 超级管理员（创始人）

**工作流程：**
1. 系统全局配置
2. 用户注册审批
3. 查看全公司利润报表
4. 审批大额采购、付款、报销
5. 查看全员绩效考核结果
6. 最终审批提成发放

**核心需求：**
- **利润核算：** 查看公司整体利润、按平台利润、按业务员利润
- **提成计算：** 最终审批所有提成，查看提成汇总报表
- **绩效考核：** 查看全员绩效汇总，进行团队整体评估

**权限需求：** 系统所有功能的访问权限

### 2.2 阿里运营

**工作流程：**
1. 阿里国际站店铺运营
2. P4P 推广费用管理
3. 询盘分配给业务员
4. 查看平台整体数据
5. 协助业务员谈单

**核心需求：**
- **利润核算：** 查看阿里平台整体利润，查看推广投入产出比
- **提成计算：** 按平台整体业绩计算运营提成
- **绩效考核：** 按询盘数量、转化率、平台销售额、ROI 考核

**权限需求：**
- 阿里平台数据完整访问
- 询盘分配权限
- 无法查看其他平台数据
- 可查看自己负责业务的利润

### 2.3 业务员（共 3 人）

**工作流程：**
1. 跟进询盘
2. 报价给客户
3. 处理样品单
4. 跟进大货订单
5. 跟进回款
6. 申请折扣、促销

**核心需求：**
- **利润核算：** 查看自己名下客户/订单的利润，实时算赚了多少钱
- **提成计算：** 实时查看自己已发提成、待发提成、预估提成
- **绩效考核：** 按销售额、回款率、新客户开发数量考核

**权限需求：**
- 只能查看自己的客户和订单
- 可以创建报价单、订单
- 需要审批才能确认折扣和大订单

### 2.4 美工（共 2 人）

**工作流程：**
1. 接收设计需求（主图、详情页、A+页面、广告图）
2. 完成设计并交付
3. 根据反馈修改
4. 管理设计素材库

**核心需求：**
- **利润核算：** 不需要直接参与利润核算
- **提成计算：** 按完成任务数量+质量评分计算提成
- **绩效考核：** 按交付及时率、客户满意度考核

**权限需求：**
- 只能查看分配给自己的设计任务
- 可以上传设计文件
- 无法查看财务数据

### 2.5 采购

**工作流程：**
1. 根据订单创建采购申请
2. 寻找供应商比价
3. 跟进采购交期
4. 跟踪验货
5. 安排物流
6. 统计供应商交货表现

**核心需求：**
- **利润核算：** 查看实际采购成本与标准成本差异，帮助降本
- **提成计算：** 按成本节约额计算绩效奖金
- **绩效考核：** 按成本节约、交期准确率、质量合格率考核

**权限需求：**
- 查看订单需求
- 创建采购申请
- 管理供应商
- 可以看采购成本但看不到售价和利润

### 2.6 出纳

**工作流程：**
1. 登记收款
2. 创建付款申请
3. 报销打款
4. 银行对账
5. 现金管理

**核心需求：**
- **利润核算：** 登记实际收支，确保利润核算准确
- **提成计算：** 提交已审批的提成打款申请
- **绩效考核：** 按差错率、及时性考核

**权限需求：**
- 只能操作收支登记
- 可以创建付款申请
- 需要财务和老板审批
- 可以看但不能修改账务数据

### 2.7 财务

**工作流程：**
1. 账务处理
2. 发票管理
3. 税务申报
4. 审核采购订单价格
5. 审核费用报销
6. 核算每个月利润
7. 汇总提成数据
8. 出具财务报表

**核心需求：**
- **利润核算：** 核心操作者，需要分摊费用、调整成本、出具利润表
- **提成计算：** 计算各角色提成，核对数据，提交审批
- **绩效考核：** 汇总各部门绩效数据，计算绩效奖金

**权限需求：**
- 完整访问财务数据
- 可以审核采购、报销、付款
- 可以创建利润报表
- 无法直接付款（出纳操作）

### 2.8 文员

**工作流程：**
1. 订单跟单
2. 物流跟踪
3. 单据录入
4. 协助安排验货
5. 整理档案

**核心需求：**
- **利润核算：** 协助录入物流、杂费等数据，保证利润准确
- **提成计算：** 不参与
- **绩效考核：** 按单据准确率、及时率考核

**权限需求：**
- 可以录入基础数据
- 无法修改价格、成本等核心财务数据
- 无法查看利润

### 2.9 亚马逊负责人

**工作流程：**
1. 亚马逊整体业务规划
2. 选品决策
3. 库存计划
4. 管理亚马逊团队
5. 审核促销、广告活动

**核心需求：**
- **利润核算：** 查看亚马逊整体利润，按 SKU 利润分析
- **提成计算：** 按团队整体业绩计算提成
- **绩效考核：** 按平台整体毛利率、ROI、增长速度考核

**权限需求：**
- 完整访问亚马逊平台所有数据
- 可以分配账号权限
- 审批亚马逊运营的促销申请
- 只能看亚马逊数据，看不到阿里数据

### 2.10 亚马逊运营

**工作流程：**
1. Listing 优化
2. 广告管理
3. 促销设置
4. 客服邮件
5. 库存监控
6. 竞品分析

**核心需求：**
- **利润核算：** 查看负责 SKU 的广告投入产出比、利润
- **提成计算：** 按销售额/利润增长计算提成
- **绩效考核：** 按广告 ACoS、销售额增长、库存周转率考核

**权限需求：**
- 只能查看负责 SKU 的数据
- 可以创建广告活动、促销（需要审批）
- 无法看财务核心数据（总成本）

## 3. 用户故事

### 超级管理员
```
作为创始人
我希望审批所有新用户注册
这样可以保证只有公司员工才能进入系统

验收标准：
- 新注册用户状态为"待审批"
- 超级管理员审批后才能登录
- 可以拒绝注册申请
```

```
作为创始人
我希望查看全公司月度利润报表
这样可以掌握整体盈利情况

验收标准：
- 按平台拆分利润
- 按业务员拆分利润
- 对比预算差异
- 支持导出 Excel
```

### 阿里运营
```
作为阿里运营
我希望给业务员分配询盘
这样可以均衡工作量，提高转化率

验收标准：
- 查看未分配询盘列表
- 分配给指定业务员
- 分配记录留存
- 业务员收到通知
```

```
作为阿里运营
我希望查看 P4P 推广 ROI
这样可以知道每个关键词投入产出比

验收标准：
- 计算 P4P 花费 ÷ 带来的销售额
- 按关键词、产品分类统计
- 识别高 ROI 关键词
```

### 业务员
```
作为业务员
我希望查看自己名下订单的实时利润
这样我知道这个单到底赚了多少钱

验收标准：
- 扣除产品成本、物流费、平台费后显示净利润
- 支持按订单查看，按客户汇总
- 数据更新不超过 5 分钟
```

```
作为业务员
我希望实时查看我的累计提成
这样我知道这个月能拿多少奖金

验收标准：
- 已回款订单计算可拿提成
- 未回款标记为待确认
- 图表展示月度对比
```

### 美工
```
作为美工
我希望查看分配给我的设计任务列表
这样我可以按优先级安排工作

验收标准：
- 显示需求描述、截止日期、需求人
- 支持标记为进行中/已完成
- 需求人收到完成通知
```

```
作为美工
我希望查看我这个月的任务数量和提成预估
这样我知道我这个月能拿多少提成

验收标准：
- 统计已完成任务数量
- 按质量评分计算预估提成
- 显示历史提成记录
```

### 采购
```
作为采购
我希望根据销售订单创建采购申请
这样可以按需采购，避免滞销

验收标准：
- 从销售订单带过来产品、数量要求
- 填写预估单价、供应商、交期
- 提交审批流程
```

```
作为采购
我希望统计每个供应商的交期准确率
这样可以帮助选择优质供应商

验收标准：
- 按月统计准时交货比例
- 延迟天数分布统计
- 供应商排名
```

### 出纳
```
作为出纳
我希望登记客户回款
这样财务可以准确核算利润

验收标准：
- 关联到对应订单
- 记录回款日期、金额、手续费
- 更新订单回款状态
```

```
作为出纳
我希望创建付款申请给供应商
这样按流程审批后打款

验收标准：
- 关联采购单
- 填写付款金额、账户信息
- 进入财务→老板审批流程
```

### 财务
```
作为财务
我希望分摊月度固定费用（房租、工资等）
这样可以准确核算净利润

验收标准：
- 按销售额比例分摊到各平台
- 支持手动调整分摊比例
- 分摊记录可追溯
```

```
作为财务
我希望计算所有人员月度提成
这样审批后可以发放

验收标准：
- 自动按规则计算业务员、美工、运营、采购提成
- 生成提成汇总表
- 提交老板审批
```

### 文员
```
作为文员
我希望录入物流跟踪信息
这样可以让业务员和客户查询物流状态

验收标准：
- 关联订单，录入快递单号
- 支持批量录入
- 状态跟踪（已发货/已签收）
```

### 亚马逊负责人
```
作为亚马逊负责人
我希望按 SKU 查看利润分析
这样可以淘汰滞销亏损产品，推广盈利产品

验收标准：
- 每个 SKU 显示销售额、成本、费用、净利润
- 计算库存周转天数
- 按毛利率排序
```

### 亚马逊运营
```
作为亚马逊运营
我希望查看广告 ACoS 和投入产出比
这样可以优化广告投放

验收标准：
- 按广告活动统计花费、销售额、ACoS
- 计算广告带来的净利润
- 支持日/周/月报表
```

### 4.1 用户注册审批流程

```
用户填写注册信息
    ↓
提交注册申请 → 状态：待审批
    ↓
超级管理员收到通知
    ↓
审核信息 → 通过/拒绝
    ↓
通过：激活账号，通知用户登录
拒绝：记录原因，通知用户
```

**验收标准：**
- 未审批用户无法登录
- 所有注册申请留痕
- 超级管理员可在后台查看待审批列表

### 4.2 数据隔离策略

| 隔离维度 | 隔离规则 |
|---------|---------|
| **按角色** | 不同角色只能看到职责范围内数据（财务才能看财务数据） |
| **按业务员** | 每个业务员只能看到自己的客户和订单 |
| **按平台** | 阿里团队看不到亚马逊数据，亚马逊团队看不到阿里数据 |
| **按 SKU（亚马逊）** | 每个亚马逊运营只能看到自己负责的 SKU |

**实现方式：**
- 数据层面：每个业务数据（客户/订单/产品）带上 `owner_id` 和 `platform` 标签
- 查询层面：自动加上过滤条件，用户无法绕过
- 权限检查：API 层面也做权限校验，前端隐藏不代表安全

## 5. 审批流程设计（6 种流程）

### 5.1 采购审批流程（金额分级）

```
采购创建申请
    ↓
金额判断
    ├─ ≤ 5000 元 → 财务审批 → 通过 → 执行
    └─ > 5000 元 → 财务初审 → 创始人审批 → 通过 → 执行

拒绝返回采购，可修改后重新提交
```

**审批节点：**
- 一级：财务（审核价格合理性）
- 二级：创始人（大额最终审批）

**金额分级：**
- P0 必须实现：5000 元分界
- 可配置：后期支持修改金额阈值

### 5.2 价格审批流程（折扣分级）

```
业务员发起折扣申请
    ↓
折扣幅度判断
    ├─ 折扣 ≤ 10% → 阿里运营/亚马逊负责人审批 → 通过
    ├─ 10% < 折扣 ≤ 20% → 负责人审批 → 财务审批 → 通过
    └─ 折扣 > 20% → → → 创始人最终审批

折扣 = (原价 - 现价) / 原价 × 100%
```

**业务场景：** 促销、老客户优惠、清库存

### 5.3 订单审批流程（大额订单）

```
业务员创建大订单
    ↓
金额判断
    ├─ ≤ 50000 元 → 自动确认
    └─ > 50000 元 → 阿里运营/亚马逊负责人审核 → 财务审核 → 创始人确认

确认后才能安排生产/采购
```

**原因：** 大额订单风险高，需要确认付款方式、利润合理性

### 5.4 付款审批流程（出纳→财务→创始人）

```
出纳创建付款申请
    ↓
财务审核（核对采购单、金额、供应商）
    ↓
金额判断
    ├─ ≤ 10000 元 → 财务审批通过 → 出纳打款
    └─ > 10000 元 → 创始人审批 → 出纳打款
```

**硬性规则：** 出纳只有执行打款权，没有审批权

### 5.5 费用报销审批流程

```
员工填写报销申请 → 上传发票照片
    ↓
部门负责人初审（确认费用因公）
    ↓
财务审核（核对发票、金额是否符合制度）
    ↓
金额判断
    ├─ ≤ 2000 元 → 财务通过 → 出纳打款
    └─ > 2000 元 → 创始人审批 → 出纳打款
```

**支持：** 差旅报销、推广费用报销、办公耗材报销

### 5.6 提成/绩效审批流程

```
财务自动计算月度提成
    ↓
部门负责人审核（阿里/亚马逊）
    ↓
财务复核
    ↓
创始人最终审批
    ↓
审批通过 → 出纳安排发放

拒绝 → 退回财务调整
```

**时间节点：** 每月 5 号前完成上月提成计算，10 号前审批完成，15 号发放

## 6. 利润核算模型

### 6.1 核心公式

**订单层面利润：**

```
订单收入 = 销售额 - 平台佣金 - 推广费用分摊 - 支付手续费
订单成本 = 产品采购成本 + 国内运费 + 国际物流费 + 报关费
订单费用 = 样品费 + 认证费 + 其他杂费分摊
订单利润 = 订单收入 - 订单成本 - 订单费用

毛利率 = 订单利润 / 订单收入 × 100%
```

**月度公司利润：**

```
月度总利润 = Σ 所有已发货订单利润 - 月度固定费用分摊
固定费用 = 房租 + 工资 + 水电 + 软件服务费 + 其他固定开支

分摊方式：按各平台销售额占比分摊
```

**平台层面利润：**

```
平台利润 = Σ 该平台所有订单利润 - 该平台直接费用（如推广年费） - 分摊固定费用
```

**业务员层面利润：**

```
业务员利润 = Σ 该业务员名下订单利润
```

**SKU 层面利润（亚马逊）：**

```
SKU 利润 = Σ SKU 销售额 - Σ SKU 采购成本 - 广告费用 - FBA 费用 - 分摊佣金
```

### 6.2 利润核算流程

```
订单创建 → 收入成本初步预估
    ↓
订单发货 → 确认实际物流费用
    ↓
客户回款 → 确认实际到账金额、手续费
    ↓
采购入库 → 确认实际采购成本
    ↓
月度结账 → 分摊固定费用
    ↓
出具最终利润报表
```

### 6.3 关键需求

1. **实时预估**：订单创建后就能看到预估利润
2. **最终核算**：所有实际费用到账后出最终利润
3. **多维度切片**：按时间、平台、业务员、SKU、客户查看利润
4. **可追溯**：每个费用项目都能看到来源，可审计

## 7. 绩效考核与提成设计

### 7.1 业务提成计算（业务员）

**支持三种计算方式，可配置：**

| 计算方式 | 公式 | 适用场景 |
|---------|------|---------|
| 按销售额 | 提成 = 销售额 × 提成比例 | 新市场开拓 |
| 按利润 | 提成 = 订单净利润 × 提成比例 | 成熟市场，关注利润 |
| 按回款 | 提成 = 实际回款金额 × 提成比例 | 控制坏账风险 |

**推荐配置：** 按利润 × 回款条件

```
业务员提成 = 已回款订单净利润 × 3% ~ 8%
阶梯提佣：
- 月度利润 < 1万 → 3%
- 1万 ≤ 月度利润 < 3万 → 5%
- 3万 ≤ 月度利润 → 8%

条件：只有回款了才能算提成
```

**验收标准：**
- 支持配置比例和阶梯
- 支持按不同业务员设置不同比例
- 未回款不计提

### 7.2 美工提成计算

**计算公式：**

```
月度提成 = 完成任务数量 × 基础单价 + 质量评分奖金 - 延迟扣分

基础单价：
- 主图：50元/张
- 详情页：300元/个
- A+页面：200元/个
- 广告图：80元/张

质量评分（由需求人评分 1~5分）：
- 5分 → +20% 奖金
- 4分 → 原价
- 3分 → -10%
- ≤2分 → 不计入，重做

延迟扣分：
- 延迟1天 → 扣10%
- 延迟3天以上 → 不计入
```

**配置化：** 支持修改各种任务类型的基础单价

### 7.3 运营绩效计算（阿里运营/亚马逊运营）

**阿里运营 KPI：**

| KPI 指标 | 权重 | 目标值 |
|---------|------|-------|
| 平台销售额 | 30% | 按月度目标 |
| 询盘数量 | 20% | 按月度目标 |
| P4P ROI | 25% | ≥ 3:1 |
| 转化率 | 15% | 目标转化率 |
| 团队协作评分 | 10% | 创始人评分 |

**绩效奖金：**
```
月度奖金 = 基准奖金 × KPI 得分百分比
得分 = Σ (实际完成 / 目标) × 权重
```

**亚马逊运营 KPI：**

| KPI 指标 | 权重 | 目标值 |
|---------|------|-------|
| 销售额增长 | 25% | 月度增长目标 |
| 广告 ACoS | 25% | ≤ 30% |
| 库存周转率 | 20% | ≥ 6次/年 |
| SKU 毛利率 | 20% | ≥ 30% |
| 客户投诉率 | 10% | < 1% |

### 7.4 采购绩效计算

| KPI 指标 | 权重 | 计算方式 |
|---------|------|---------|
| 成本节约额 | 40% | (目标成本 - 实际成本) × 采购量 |
| 交期准确率 | 30% | 准时交货订单数 ÷ 总订单数 × 100% |
| 质量合格率 | 20% | 合格批次 ÷ 总批次 × 100% |
| 供应商开发数量 | 10% | 月度新开发合格供应商数 |

**绩效奖金：**
```
采购奖金 = 成本节约额 × 10% + 绩效评分 × 基础奖金

即：采购帮公司省下的钱，拿 10% 作为奖金，激励降本
```

### 7.5 其他角色绩效考核

**出纳/财务/文员：**

| KPI | 权重 |
|-----|------|
| 工作差错率 | 40% |
| 完成及时性 | 30% |
| 创始人评分 | 30% |

**固定薪资 + 季度奖金，按评分发放**

## 9. 数据隔离策略

### 9.1 三层隔离架构

```
第一层：按平台隔离
  ├─ 阿里国际站
  ├─ 亚马逊
  ├─ TikTok
  └─ 独立站
  规则：平台负责人只能看本平台数据

第二层：按用户/业务员隔离
  └─ 每个业务员只能看自己的客户和订单

第三层：按角色权限隔离
  ├─ 业务员看不到成本结构
  ├─ 采购看不到售价和利润
  ├─ 财务才能看全财务数据
```

### 9.2 实现方案

**数据库层面：**
- 所有业务表都有 `platform_id` 字段
- 所有业务表都有 `owner_user_id` 字段
- 查询时自动附加 `WHERE platform_id = ? AND owner_user_id = ?`（根据用户权限）

**API 层面：**
- 每个 API 都做权限校验
- 即使用户手动改 ID 也无法访问别人的数据

**前端层面：**
- 没有权限的菜单和按钮不显示
- 敏感字段（成本）对无权限用户隐藏
