## P1 - 必须完成

---

### US-001: 用户登录

**作为** 系统用户  
**我想要** 通过邮箱和密码登录系统  
**以便** 访问我有权限的功能

**优先级**: 🔴 P1  
**故事点**: 3 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-001-用户登录成功)

**验收要点**:
- [ ] 正确用户名密码可以登录成功
- [ ] 错误密码返回 401
- [ ] 登录成功设置 HttpOnly Cookie
- [ ] 生产环境自动添加 Secure 标志

**依赖**: 无

---

### US-002: 保持登录

**作为** 系统用户  
**我想要** 刷新页面后保持登录状态  
**以便** 不需要重复输入密码

**优先级**: 🔴 P1  
**故事点**: 1 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-015-刷新页面保持登录)

**验收要点**:
- [ ] 刷新页面后仍然是登录状态
- [ ] 不需要重新输入密码

**依赖**: US-001

---

### US-003: 用户登出

**作为** 系统用户  
**我想要** 可以主动登出系统  
**以便** 在公共设备上保护账户安全

**优先级**: 🔴 P1  
**故事点**: 1 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-008-用户登出)

**验收要点**:
- [ ] 登出成功清除 Cookie
- [ ] 登出后访问需要认证的 API 返回 401

**依赖**: US-001

---

### US-004: 获取当前用户

**作为** 前端开发者  
**我想要** 能够获取当前登录用户的信息  
**以便** 根据用户信息显示个性化内容

**优先级**: 🔴 P1  
**故事点**: 1 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-006-获取当前用户---已登录)

**验收要点**:
- [ ] 已登录返回用户信息（id, email, name, role）
- [ ] 未登录返回 401

**依赖**: US-001

---

### US-005: API 认证检查

**作为** 后端开发者  
**我想要** 在 API 中方便地验证用户登录状态  
**以便** 保护需要认证的接口

**优先级**: 🔴 P1  
**故事点**: 2 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-011-权限验证---已认证通过)

**验收要点**:
- [ ] 已认证用户可以访问
- [ ] 未认证用户返回 401
- [ ] 调用方式简单：`const authError = await requireAuth(request)`

**依赖**: US-001, US-004

---

### US-006: API 权限检查

**作为** 后端开发者  
**我想要** 在 API 中方便地验证用户角色权限  
**以便** 确保只有有权限的用户才能执行敏感操作

**优先级**: 🔴 P1  
**故事点**: 2 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-013-角色验证---权限足够通过)

**验收要点**:
- [ ] 支持四种角色：VIEWER < USER < MANAGER < ADMIN
- [ ] 高等级角色可以访问低等级资源
- [ ] 权限不足返回 403
- [ ] 调用方式简单：`const roleError = requireRole(session, requiredRole)`

**依赖**: US-005

---

### US-007: 防止暴力破解

**作为** 系统管理员  
**我想要** 限制登录失败次数  
**以便** 防止攻击者暴力破解密码

**优先级**: 🔴 P1  
**故事点**: 2 SP  
**验收标准**: [链接到验收标准](../requirements/v0.7.0-ACCEPTANCE.md#ac-009-速率限制---超过-5-次失败)

**验收要点**:
- [ ] 15 分钟内最多允许 5 次登录失败
- [ ] 超过限制返回 429 Too Many Requests
- [ ] 返回 Retry-After 头
- [ ] 锁定时间过后自动解锁

**依赖**: US-001

---

### US-008: 统一认证方案

**作为** 开发者  
**我想要** 项目只使用一种认证方案  
**以便** 降低维护成本，减少混淆

**优先级**: 🔴 P1  
**故事点**: 3 SP  
**验收标准**:
- [ ] 所有 API 统一使用简化认证
- [ ] 统一从 `@/lib/auth` 导出认证函数
- [ ] 消除双方案维护成本

**依赖**: 所有上述故事

## P2 - 后续迭代

---

### US-009: 账户状态控制

**作为** 系统管理员  
**我想要** 不同状态的账户有不同的登录提示  
**以便** 告知用户账户当前状态

**优先级**: 🟡 P2  
**故事点**: 1 SP  
**验收标准**:
- [ ] PENDING_APPROVAL: 返回"账户等待审批"
- [ ] SUSPENDED: 返回"账户已暂停"
- [ ] DISABLED: 返回"账户已禁用"

**依赖**: US-001

---

### US-010: 密码强度验证

**作为** 系统管理员  
**我想要** 强制用户设置强密码  
**以便** 提高账户安全性

**优先级**: 🟡 P2  
**故事点**: 2 SP  
**验收标准**:
- [ ] 密码长度至少 8 位
- [ ] 要求包含大小写字母和数字
- [ ] 注册时验证密码强度
- [ ] 弱密码拒绝注册并提示

**依赖**: 无（用户注册功能）

---

### US-011: 登录审计日志

**作为** 系统管理员  
**我想要** 记录所有登录/登出事件  
**以便** 出现安全问题可以追踪

**优先级**: 🟡 P2  
**故事点**: 2 SP  
**验收标准**:
- [ ] 记录登录时间、IP、用户代理
- [ ] 记录登录成功/失败
- [ ] 记录登出时间
- [ ] 管理员可以查看审计日志

**依赖**: US-001, US-003
