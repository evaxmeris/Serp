## 1. 用户需求深度分析

### 1.1 各角色日常工作流程

#### 1.1.1 创始人 / 超级管理员

**日常工作：**
- 查看整体业务数据报表（营收、利润、库存）
- 审批大额采购、付款、报销
- 审批特殊折扣价格
- 审核新品开发计划
- 管理团队权限和用户账号
- 对接核心供应商谈判
- 财务总账审核
- 战略方向制定

**每周/每月：**
- 月度业绩复盘
- 制定下月销售目标
- 供应商付款计划审批
- 工资核算审批

### 1.2 痛点与效率瓶颈识别

| 角色 | 痛点描述 | 影响程度 | 建议解决方案 |
|------|----------|----------|--------------|
| **创始人** | 数据分散在各个平台，需要手动汇总报表，耗时且容易出错 | 🔴 高 | 统一数据看板，自动生成多维度报表 |
| | 审批流程不规范，微信/邮件审批容易遗漏 | 🔴 高 | 系统内置审批流，实时提醒 |
| **运营经理** | 库存不准确，超卖断货时有发生 | 🔴 高 | 多平台库存同步，自动预警 |
| | 多平台数据需要分别登录查看，操作繁琐 | 🟡 中 | 统一后台多平台订单同步 |
| **业务员** | 客户信息分散在Excel，跟进不及时丢单 | 🔴 高 | CRM 客户管理，跟进提醒 |
| | 报价单需要手动计算价格，容易出错 | 🟡 中 | 系统自动计算，模板导出 |
| **采购** | 采购价格历史无记录，难以议价对比 | 🔴 高 | 供应商价格库，历史趋势分析 |
| | 海外仓备货计划靠经验，容易积压或缺货 | 🔴 高 | 基于销售数据智能备货建议 |
| **美工** | 设计文件管理混乱，找图耗时间 | 🟡 中 | 素材资源库管理，按产品分类 |
| **出纳** | 付款需要核对审批流程，手工登账容易错 | 🔴 高 | 系统记录审批状态，自动生成流水 |
| | 银行水单归档零散，对账难找 | 🟡 中 | 水单电子化归档，关联对应付款 |
| **财务** | 利润核算需要从多个部门收集数据，耗时 | 🔴 高 | 自动按订单/产品核算利润 |
| | 应收款回款靠业务员跟进，逾期提醒不及时 | 🟡 中 | 系统自动逾期提醒 |
| **文员** | 单证文件存放混乱，需要时找不到 | 🟡 中 | 按订单归档电子文件 |
| **亚马逊负责人** | FBA库存和国内库存不同步，容易断货 | 🔴 高 | 统一库存管理，FBA库存实时同步 |
| **全体** | 多平台需要分别登录，账号密码管理麻烦 | 🟢 低 | SSO 单点登录 |

**核心痛点总结：**
1. **数据孤岛**：各平台数据分散，无法统一查看
2. **流程不规范**：审批无记录，依赖人工跟进容易遗漏
3. **财务断层**：业务数据和财务数据脱节，利润核算困难
4. **库存不准确**：多平台库存不同步，超卖/断货
5. **客户流失**：客户信息未系统化管理，跟进不及时
6. **采购盲猜**：备货靠经验，缺乏数据支撑

### 1.3 多平台业务差异分析

| 维度 | 阿里国际站（现有） | 亚马逊（规划） | TikTok（规划） | 独立站（规划） |
|------|-------------------|---------------|---------------|----------------|
| **订单类型** | 大额批发 (MOQ) | FBA/自发货 小额多单 | 小额批发/DTC | DTC 零售 |
| **付款方式** | T/T 信用证 | 平台收款 | PayPal/信用卡 | PayPal/信用卡 |
| **价格体系** | 阶梯报价 | 固定定价 | 活动定价 | 动态定价 |
| **库存管理** | 国内仓直发为主 | FBA 海外仓 | 海外仓一件代发 | 海外仓一件代发 |
| **客户管理** | B 端企业客户 | 终端消费者 | 混合 | 终端消费者 |
| **售后** | 较少纠纷 | 退换货频繁 | 退换货多 | 退换货多 |
| **对账周期** | 月结/款到发货 | 周期结算 | 周期结算 | 周期结算 |
| **核心需求** | 询盘转化、CRM | FBA 库存管理 | 内容标签同步 | 订单同步 |

**关键差异总结：**
- **B 端 vs C 端**：阿里国际站是 B 端批发，其他三个偏向 C 端/小 B
- **仓储模式**：阿里以国内直发为主，亚马逊/TikTok/独立站需要海外仓一件代发
- **订单规模**：阿里单量大、单量少；其他平台单量多、单量小
- **价格策略**：B 端需要阶梯报价、折扣审批；C 端需要灵活定价

### 1.4 多平台业务协同需求

#### 1.4.1 库存协同

**需求：**
- 统一库存池：国内仓 + 多个海外仓统一管理
- 多平台扣减：任一平台下单，对应仓库库存自动扣减
- 库存预警：各仓库分别设置安全库存，低于安全线自动提醒补货
- FBA 补货：根据亚马逊销量自动计算FBA补货建议

**业务流程：**
```
用户下单 → 判断发货仓库 → 扣减对应仓库库存 → 库存低于安全线 → 提醒补货
```

#### 1.4.2 订单协同

**需求：**
- 多平台订单自动拉取到统一后台
- 同一仓库的订单可以合并发货
- 物流信息回写到对应平台
- 财务按平台分别统计营收

#### 1.4.3 产品协同

**需求：**
- 基础产品信息一份维护，多平台同步
- 每个平台可以设置不同的售价
- 图片素材共享，多平台复用
- 库存共享，成本统一核算

#### 1.4.4 财务协同

**需求：**
- 按平台分别核算营收、费用、利润
- 平台手续费自动扣减核算
- 多平台回款统一对账
- 汇率自动转换计算

#### 1.4.5 供应链协同

**需求：**
- 汇总多平台销售数据，统一生成采购计划
- 海外仓备货需求纳入整体采购计划
- 采购成本按产品分摊，准确计算各平台产品利润

### 2.2 用户注册审批流程

```
用户提交注册申请
        ↓
超级管理员收到审批提醒
        ↓
超级管理员审核：
  ✅ 通过 → 分配角色 → 账号激活
  ❌ 拒绝 → 填写拒绝理由 → 通知申请人
        ↓
完成
```

**规则：**
- 新用户必须经过超级管理员审批才能激活
- 支持批量审批
- 保留审批记录日志
- 支持邀请注册链接（超级管理员生成）
- 角色变更也需要超级管理员审批

### 2.4 数据隔离策略

#### 2.4.1 按角色隔离

**核心原则：** 只看到自己工作需要的数据，敏感数据对无关人员隐藏

| 数据类型 | 可访问角色 | 不可访问角色 |
|----------|-----------|-------------|
| 客户信息（全公司） | 创始人、运营 | 业务员只能看自己的，其他人不可见 |
| 采购成本 | 创始人、运营、采购、财务 | 业务员、美工、出纳不可见 |
| 公司整体财务数据 | 创始人、财务 | 其他角色不可见 |
| 资金流水 | 创始人、出纳、财务 | 其他角色不可见 |
| 业务员业绩 | 创始人、运营、本人 | 其他业务员不可见 |
| 亚马逊全店数据 | 创始人、运营、亚马逊负责人 | 其他业务员不可见 |

#### 2.4.2 按业务员隔离（客户数据）

**策略：** 基于负责人隔离
- 业务员只能查看、编辑自己负责的客户
- 运营经理和超级管理员可以查看所有客户
- 支持客户分配转移，转移后原负责人无法查看
- 支持团队共享客户（可选配置，由运营开启）

#### 2.4.3 按平台隔离

**策略：**
- 亚马逊负责人只能查看亚马逊平台数据
- 阿里国际站数据由对应业务员管理
- 财务可以查看所有平台财务数据
- 库存数据统一共享，但按平台统计销量

#### 2.4.4 敏感数据分级

| 敏感级别 | 数据举例 | 访问权限 |
|----------|---------|---------|
| 🔴 极高 | 公司整体财务报表、利润、银行余额 | 创始人 + 财务 |
| 🟠 高 | 采购成本、供应商价格 | 创始人 + 运营 + 采购 + 财务 |
| 🟡 中 | 客户信息、订单详情 | 负责人 + 上级管理 |
| 🟢 低 | 产品基础信息（非成本）、公共素材 | 全体可访问 |

## 3. 外贸审批流程设计

### 3.1 采购审批（金额分级）

**分级标准：**

| 采购金额 | 审批流程 | 审批人 |
|----------|----------|--------|
| < 5000 元 | 采购申请 → 运营经理审批 → 完成 | 运营经理 |
| 5000 - 50000 元 | 采购申请 → 运营经理初审 → 创始人审批 → 完成 | 运营经理 → 创始人 |
| > 50000 元 | 采购申请 → 运营经理初审 → 创始人终审 → 完成 | 创始人 |

**特殊情况：**
- FBA 补货采购：亚马逊负责人发起申请 → 运营审核 → 按金额分级审批
- 样品采购：< 1000 元 → 运营直接审批

**流程图：**

```
采购/亚马逊负责人创建采购申请
        ↓
系统判断金额区间
        ↓
< 5000 → 运营审批 → 通过 → 执行采购
                  ↓
                 拒绝 → 退回给申请人
        ↓
5000 - 50000 → 运营审批 → 通过 → 创始人审批 → 通过 → 执行采购
                                  ↓
                             拒绝 → 退回
        ↓
> 50000 → 运营预审 → 创始人审批 → 通过 → 执行采购
```

**业务规则：**
- 采购不能审批自己发起的申请
- 支持审批前修改撤回
- 支持添加审批意见
- 超期未审批自动提醒（钉钉/消息）
- 所有审批记录永久留存

---

### 3.2 价格审批（折扣权限）

**折扣分级：**

| 折扣幅度 | 相对于标准价 | 审批人 |
|----------|-------------|--------|
| 折扣 ≥ 95 折 | 优惠 ≤ 5% | 业务员自动通过 |
| 85 折 - 94 折 | 优惠 6%-15% | 运营经理审批 |
| < 85 折 | 优惠 > 15% | 创始人审批 |

**特殊情况：**
- 老客户返单折扣可由运营审批
- 样品单折扣规则同上
- 大MOQ批量订单可以申请更大折扣，必须创始人审批
- 亚马逊促销折扣由亚马逊负责人发起 → 运营审批

**流程图：**

```
业务员创建报价
        ↓
计算折扣幅度
        ↓
≥ 95 折 → 自动通过 → 可发送给客户
        ↓
85-94 折 → 运营经理审批 → 通过 → 发送客户
                        ↓
                       拒绝 → 退回修改
        ↓
< 85 折 → 运营初审 → 创始人审批 → 通过 → 发送客户
```

---

### 3.3 订单审批（大额订单）

**定义：** 大额订单 = 单订单金额 > 100,000 元

**审批流程：**

```
业务员创建订单
        ↓
判断订单金额
        ↓
≤ 100,000 → 自动通过 → 安排采购/生产
        ↓
> 100,000 → 创始人审批 → 通过 → 安排生产
                        ↓
                       拒绝 → 退回业务员
```

**业务规则：**
- 收到预付款的大订单可直接通过，无需审批
- 信用证付款的大订单必须审批
- 新客户首单大额必须审批
- 亚马逊B2B大额订单同样遵循此规则

---

### 3.4 付款审批（分级授权）

**金额分级：**

| 付款金额 | 审批流程 | 审批人 |
|----------|----------|--------|
| < 10000 元 | 付款申请 → 运营经理审核 → 财务复核 → 出纳付款 | 运营经理 + 财务 |
| 10000 - 100000 元 | 付款申请 → 运营初审 → 财务复核 → 创始人审批 → 出纳付款 | 运营 → 财务 → 创始人 |
| > 100000 元 | 付款申请 → 运营初审 → 财务复核 → 创始人终审 → 出纳付款 | 创始人 |

**流程图：**

```
采购创建付款申请
  ↓
  附上对账单/发票
  ↓
财务初审（核对账款是否一致）
  ↓
金额判断
  ↓
< 10000 → 运营审批 → 通过 → 出纳付款
                    ↓
                   拒绝 → 退回
  ↓
≥ 10000 → 运营初审 → 通过 → 创始人审批 → 通过 → 出纳付款
```

**业务规则：**
- 付款必须附件对应对账单/发票（支持上传PDF/JPG）
- 同一供应商月度合并付款，按总金额走审批
- 紧急付款可标记加急，优先提醒审批人
- 支持批量付款申请，按单笔金额分别走审批
- 出纳只能在审批完成后才能付款，不允许提前付款

**特殊规则：**
- 运费、货代费用由采购申请，< 10000 运营+财务审批
- 样品费小额采购走快速审批通道（运营审批）

### 3.5 费用报销审批

**报销分类：**

| 报销金额 | 审批流程 | 审批人 |
|----------|----------|--------|
| < 1000 元 | 申请人提交 → 运营经理审批 → 财务复核 → 出纳打款 | 运营 + 财务 |
| 1000 - 5000 元 | 申请人提交 → 运营经理审核 → 财务复核 → 创始人审批 → 出纳打款 | 运营 → 财务 → 创始人 |
| > 5000 元 | 申请人提交 → 运营经理审核 → 财务复核 → 创始人终审 → 出纳打款 | 创始人 |

**报销类型：**
- 差旅费
- 业务招待费
- 办公用品
- 广告服务费
- 平台费用
- 其他

**业务规则：**
- 必须上传发票照片/PDF

- 出纳在审批完成后 1-2 个工作日打款
- 所有报销记录归档，可随时查询

**流程图：**

```
员工填写报销申请
  ↓
  上传发票附件
  ↓
  提交
  ↓
< 1000 → 运营审批 → 通过 → 财务复核 → 通过 → 出纳付款
        ↓
≥ 1000 → 运营审核 → 通过 → 财务复核 → 通过 → 创始人审批 → 通过 → 出纳付款
```

## 6. 验收标准

### 6.1 功能验收标准

| 验收项 | 验收标准 | 测试方法 |
|--------|----------|----------|
| **权限隔离** | 业务员登录后只能看到自己的客户，看不到同事的；财务能看到财务数据，业务员看不到成本 | 使用不同角色账号登录，尝试越权访问 | P0 |
| **9角色权限** | 每个角色只能看到权限矩阵中允许的功能，不能看到未授权功能 | 用每个角色登录，检查菜单可见性 | P0 |
| **分级审批** | 不同金额走不同审批流，正确到达对应审批人 | 创建不同金额申请，检查审批节点是否正确 | P0 |
| **折扣审批** | 折扣超过权限自动进入审批，不允许直接发送 | 创建不同折扣报价，检查是否正确触发审批 | P0 |
| **付款审批流程** | 付款申请必须经过运营→财务→创始人分级，出纳只能看到已审批付款 | 创建不同金额付款申请，检查流程 | P0 |
| **报销审批流程** | 报销按金额分级审批，出纳只能付已审批报销 | 创建不同金额报销，检查流程 | P0 |
| **库存更新** | 采购入库后库存数量正确更新，多平台下单正确扣减对应仓库 | 创建采购单，入库前后检查库存变化；下单后检查库存 | P0 |
| **多平台库存协同** | 亚马逊下单扣减FBA库存，阿里下单扣减国内库存，互不干扰 | 在不同平台创建订单，检查对应仓库库存变化 | P0 |
| **利润核算** | 订单利润 = 收入 - 采购成本 - 运费 - 手续费，计算正确 | 手动计算对比系统计算结果 | P0 |
| **数据报表** | 报表数据和明细数据对得上，统计正确 | 对比报表汇总和订单明细汇总 | P0 |
| **审批提醒** | 有新审批时，审批人能收到通知 | 创建审批，检查审批人是否收到提醒 | P0 |
| **文件上传** | 支持PDF/图片附件上传，可下载查看 | 上传测试文件，尝试下载 | P0 |
| **数据导出** | 支持Excel/PDF导出，数据完整 | 导出报表，检查数据完整性 | P0 |

### 6.2 性能验收标准

- 页面响应时间 < 2 秒
- 支持 15 人同时在线使用无明显卡顿
- 数据导出在 1 分钟内完成
- 支持 10 万+ 客户数据存储

### 6.3 安全验收标准

- 密码加密存储
- 操作日志留存 1 年以上
- 敏感数据（成本、财务）按权限隐藏
- 越权访问测试不能通过
- URL直接访问未授权页面返回403
