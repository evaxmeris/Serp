## 1. 项目背景

### 1.1 业务背景

随着跨境电商业务的发展，当前 Trade ERP 系统面临以下挑战：

1. **多用户协作需求**：业务扩展后需要多个运营人员、客服、管理员协同操作，但缺乏细粒度的权限控制
2. **多平台业务拓展**：除了现有亚马逊平台，业务已拓展至阿里国际站，需要统一订单管理
3. **多仓库布局**：国内仓+海外仓的配送体系已建立，需要系统支持多仓库库存管理
4. **效率提升需求**：产品需要同时刊登到多个销售渠道，手动操作效率低下

### 1.2 项目目标

本次 v0.8.0 版本旨在解决以上问题，实现：

- ✅ 完善的权限控制体系，支持不同角色不同权限
- ✅ 阿里国际站订单自动同步，统一管理多平台订单
- ✅ 支持国内仓+海外仓多仓库存管理
- ✅ 一次上传，多平台产品自动刊登

### 1.3 目标用户

| 用户角色 | 主要功能需求 |
|---------|-------------|
| 超级管理员 | 系统配置、用户管理、全权限访问 |
| 运营经理 | 订单管理、产品管理、报表查看 |
| 运营专员 | 订单处理、产品刊登、库存查询 |
| 客服人员 | 订单查询、客户咨询处理 |
| 仓库管理员 | 库存管理、出入库操作 |

## 2. 功能需求详细描述

### 2.1 RBAC 权限管理

#### 2.1.1 需求概述

基于 RBAC (Role-Based Access Control) 模型实现用户-角色-权限三级控制，支持灵活的权限分配。

#### 2.1.2 功能模块

##### 2.1.2.1 用户管理

- **功能描述**：管理系统用户账号
- **需求点**：
  - 支持创建用户（填写用户名、邮箱、密码、分配角色）
  - 支持编辑用户信息（重置密码、启用/禁用账号）
  - 支持删除用户（软删除，保留历史数据）
  - 支持用户列表查看（分页、搜索）
  - 支持查看用户登录日志

##### 2.1.2.2 角色管理

- **功能描述**：定义角色并分配权限
- **需求点**：
  - 支持创建角色（角色名称、角色描述）
  - 支持编辑角色权限
  - 支持删除角色（无用户绑定才可删除）
  - 内置默认角色：超级管理员、运营经理、运营专员、客服、仓库管理员
  - 支持自定义角色

##### 2.1.2.3 权限管理

- **功能描述**：系统权限项定义
- **需求点**：
  - 权限按模块分组：系统管理、订单管理、产品管理、库存管理、报表查看
  - 支持细粒度控制：可查看/可编辑/可删除不同级别
  - 权限项自动扫描注册
  - 支持权限勾选分配给角色

##### 2.1.2.4 访问控制

- **功能描述**：API 层面权限拦截
- **需求点**：
  - 用户登录后获取角色权限
  - API 接口权限校验，无权限返回 403
  - 前端页面根据权限控制按钮/菜单显示隐藏
  - 支持超级管理员绕过所有权限检查

#### 2.1.3 数据模型

```
User
├── id: UUID
├── username: String (unique)
├── email: String (unique)
├── passwordHash: String
├── isActive: Boolean
├── createdAt: DateTime
├── updatedAt: DateTime
└── roles: Role[] (many-to-many)

Role
├── id: UUID
├── name: String (unique)
├── description: String
├── isSystem: Boolean (内置角色不可删除)
├── createdAt: DateTime
├── updatedAt: DateTime
└── permissions: Permission[] (many-to-many)

Permission
├── id: UUID
├── code: String (unique, e.g. "order:read")
├── name: String
├── module: String ("system" | "order" | "product" | "inventory")
├── description: String
└── createdAt: DateTime
```

### 2.2 跨境电商多平台接入

#### 2.2.1 需求概述

对接多个跨境电商平台 API，实现订单自动同步到 ERP 系统统一管理。**优先对接阿里国际站**。

#### 2.2.2 功能模块

##### 2.2.2.1 平台授权管理

- **功能描述**：管理各个平台的授权信息
- **需求点**：
  - 支持添加平台授权（选择平台类型、填入 AppKey/AppSecret/AccessToken）
  - 支持测试授权连接
  - 支持刷新令牌（当 token 过期时自动刷新）
  - 支持删除授权
  - 授权列表展示，显示授权状态

##### 2.2.2.2 阿里国际站订单同步

- **功能描述**：从阿里国际站同步订单到 ERP
- **需求点**：
  - 支持手动触发单次同步
  - 支持定时自动同步（可配置同步间隔，默认 15 分钟）
  - 支持同步全部订单或按时间范围同步
  - 订单状态映射：
    - 等待买家付款 → `pending_payment`
    - 买家已付款 → `paid`
    - 待发货 → `pending_fulfillment`
    - 已发货 → `shipped`
    - 交易完成 → `completed`
    - 交易取消 → `cancelled`
  - 同步信息包括：
    - 订单基本信息（订单号、下单时间、买家信息）
    - 商品信息（SKU、数量、单价）
    - 收货地址信息
    - 支付金额、运费、手续费
    - 物流信息
  - 订单已存在则更新，不存在则新建
  - 同步完成后显示同步结果统计（成功数、失败数）

##### 2.2.2.3 订单状态回传

- **功能描述**：将 ERP 内发货状态回传到阿里国际站
- **需求点**：
  - 支持手动触发发货信息回传
  - 支持自动回传（ERP 发货后自动同步到平台）
  - 回传信息包括：物流服务商、运单号、发货时间

##### 2.2.2.4 多平台订单统一视图

- **功能描述**：在 ERP 内查看管理所有平台订单
- **需求点**：
  - 订单列表显示来源平台标识
  - 支持按平台筛选订单
  - 保持现有订单管理功能不变，新增平台订单兼容处理

##### 2.2.2.5 同步日志

- **功能描述**：记录平台同步操作日志，方便排查问题
- **需求点**：
  - 记录每次同步开始/结束时间
  - 记录成功/失败数量
  - 记录具体错误信息（当同步失败时）
  - 支持查看同步日志详情

#### 2.2.3 支持平台计划

| 平台 | 优先级 | 状态 |
|-----|--------|------|
| 阿里国际站 | P0 | ✅ 本次实现 |
| 亚马逊 | P1 | 后续版本 |
| eBay | P2 | 后续版本 |
| Shopify | P2 | 后续版本 |
| TikTok Shop | P1 | 后续版本 |

### 2.3 多仓库管理

#### 2.3.1 需求概述

支持多个仓库（国内仓、海外仓）的库存管理，每个仓库独立库存计算，支持库存调拨。

#### 2.3.2 功能模块

##### 2.3.2.1 仓库管理

- **功能描述**：维护仓库信息
- **需求点**：
  - 支持创建仓库（仓库名称、仓库编码、仓库类型、地址、联系人、电话）
  - 仓库类型：国内仓 / 海外仓
  - 支持编辑仓库信息
  - 支持停用/启用仓库
  - 支持设置默认仓库（新建订单默认带出）
  - 仓库列表展示

##### 2.3.2.2 库存按仓库分离

- **功能描述**：每个仓库独立计算库存
- **需求点**：
  - SKU 库存按仓库分开存储
  - 库存变动记录（出入库）关联仓库
  - 订单扣减库存时指定扣减哪个仓库的库存
  - 支持查看每个仓库的库存明细

##### 2.3.2.3 库存调拨

- **功能描述**：在多个仓库之间调拨库存
- **需求点**：
  - 支持创建调拨单（调出仓库、调入仓库、SKU、数量、备注）
  - 调拨单状态：待审核 / 已出库 / 已入库 / 已取消
  - 审核通过后，扣减调出仓库库存，增加调入仓库在途库存
  - 入库完成后，增加调入仓库可用库存，减少在途库存
  - 支持查看调拨单列表
  - 库存调拨产生库存变动记录

##### 2.3.2.4 出入库管理

- **功能描述**：仓库出入库操作
- **需求点**：
  - 采购入库：选择仓库，增加库存
  - 退货入库：选择仓库，增加库存
  - 盘盈入库：选择仓库，增加库存
  - 盘亏出库：选择仓库，减少库存
  - 损毁出库：选择仓库，减少库存
  - 每笔出入库生成库存变动记录

##### 2.3.2.5 库存预警

- **功能描述**：低库存预警
- **需求点**：
  - 支持为 SKU 设置每个仓库的安全库存
  - 当可用库存低于安全库存时，系统标记为低库存
  - 支持查看低库存 SKU 列表

##### 2.3.2.6 仓库关联发货

- **功能描述**：订单发货时选择从哪个仓库发出
- **需求点**：
  - 订单处理环节支持选择发货仓库
  - 默认带出默认仓库
  - 发货后自动扣减对应仓库库存

#### 2.3.3 数据模型变更

现有 `Inventory` 模型新增 `warehouseId` 字段，按仓库分库存。

```
Warehouse
├── id: UUID
├── name: String
├── code: String (unique)
├── type: Enum ("domestic" | "overseas")
├── address: String?
├── contactName: String?
├── contactPhone: String?
├── isDefault: Boolean
├── isActive: Boolean
├── createdAt: DateTime
└── updatedAt: DateTime

InventoryTransfer
├── id: UUID
├── transferNo: String (unique)
├── fromWarehouseId: UUID
├── toWarehouseId: UUID
├── status: Enum ("pending" | "outbound" | "inbound" | "cancelled")
├── remark: String?
├── createdAt: DateTime
└── updatedAt: DateTime

InventoryTransferItem
├── id: UUID
├── transferId: UUID
├── productSkuId: UUID
├── quantity: Integer
└── createdAt: DateTime
```

### 2.4 多平台产品刊登

#### 2.4.1 需求概述

在 ERP 维护产品信息，一键刊登到多个销售平台，避免重复录入。

#### 2.4.2 功能模块

##### 2.4.2.1 刊登模板管理

- **功能描述**：为不同平台配置刊登模板
- **需求点**：
  - 支持按平台创建刊登模板
  - 模板映射：ERP 产品字段 → 平台字段映射
  - 支持配置默认分类（刊登到平台哪个类目）
  - 支持编辑模板
  - 支持删除模板（未使用才可删除）

##### 2.4.2.2 产品一键刊登

- **功能描述**：将 ERP 产品刊登到指定平台
- **需求点**：
  - 在产品详情页支持选择平台发起刊登
  - 支持批量选择产品批量刊登
  - 根据模板自动填充平台产品信息
  - 调用平台 API 创建产品
  - 返回刊登结果（成功/失败）
  - 失败显示错误原因
  - 刊登成功后，记录平台商品 ID 关联到 ERP 产品

##### 2.4.2.3 刊登任务管理

- **功能描述**：查看管理所有刊登任务
- **需求点**：
  - 刊登任务列表展示
  - 显示任务状态：待处理 / 处理中 / 成功 / 部分成功 / 失败
  - 支持查看任务详情
  - 支持重试失败的刊登任务

##### 2.4.2.4 价格换算

- **功能描述**：根据成本和利润率自动计算平台售价
- **需求点**：
  - 支持设置利润率公式
  - 支持按平台不同货币换算
  - 自动计算建议售价

##### 2.4.2.5 产品信息同步

- **功能描述**：从平台同步产品信息到 ERP
- **需求点**：
  - 支持从平台拉取现有产品信息
  - 支持增量同步和全量同步
  - 创建或更新 ERP 产品信息

##### 2.4.2.6 刊登优先级

优先支持阿里国际站产品刊登，后续版本支持其他平台。

## 4. 验收标准

### 4.1 RBAC 权限管理验收标准

#### 功能验收

| 验收项 | 验收标准 | 状态 |
|-------|---------|------|
| 用户创建 | 可以成功创建用户，用户名邮箱唯一校验，密码加密存储 | ☐ |
| 用户禁用 | 禁用用户后，该用户无法登录 | ☐ |
| 角色创建 | 可以创建自定义角色，并分配权限 | ☐ |
| 内置角色 | 系统内置 5 个默认角色，不可删除 | ☐ |
| 权限拦截 | 无权限用户访问 API 返回 403，前端不显示对应菜单按钮 | ☐ |

## 6. 技术考量

### 6.1 API 集成

- 阿里国际站开放平台使用 OAuth 2.0 授权
- 需要处理 API 限流，实现合理退避重试
- 同步任务异步处理，避免超时

### 6.2 数据一致性

- 库存变动需要使用事务保证数据一致性
- 订单同步需要实现幂等，避免重复创建
- 分仓库存扣减需要原子操作

### 6.3 性能

- 定时同步订单使用队列异步处理
- 权限检查可以缓存，减少数据库查询
- 库存查询合理添加索引

### 6.4 扩展性

- 多平台接入使用适配器模式，便于后续新增平台
- 权限模型设计支持新增权限项不用改结构
- 多仓库设计支持无限扩展仓库数量
