## 🧪 测试策略

### 测试类型

#### 1. 单元测试 (Unit Testing)
- **目标**: 验证单个函数/方法的正确性
- **工具**: Jest / Vitest
- **覆盖范围**: 
  - 属性值验证函数
  - 对比算法函数
  - 数据转换函数
- **负责人**: 开发团队

#### 2. 集成测试 (Integration Testing)
- **目标**: 验证模块间接口和数据流
- **工具**: Jest + Supertest
- **覆盖范围**:
  - API 端点测试
  - 数据库操作测试
  - 模块间依赖测试（调研→备货→采购）
- **负责人**: QA 团队

#### 3. 端到端测试 (E2E Testing)
- **目标**: 验证完整用户流程
- **工具**: Playwright
- **覆盖范围**:
  - 创建品类→属性模板→产品调研完整流程
  - 产品对比完整流程
  - 调研转备货计划完整流程
- **负责人**: QA 团队

#### 4. API 测试
- **目标**: 验证 REST API 功能
- **工具**: Jest + Supertest
- **覆盖范围**: 所有产品调研相关 API
- **负责人**: QA 团队

#### 5. 性能测试
- **目标**: 验证系统在高负载下的表现
- **工具**: k6 / Artillery
- **覆盖范围**:
  - 多产品对比性能（10+ 产品）
  - 大数据量查询
- **负责人**: 运维团队

- [ ] 所有 P0/P1 测试用例执行完毕

- [ ] 性能指标达标（API < 200ms）

### 缺陷优先级定义

| 优先级 | 定义 | 修复时限 |
|--------|------|----------|
| **P0 - 严重** | 系统崩溃、数据丢失、核心功能不可用 | 24 小时 |
| **P1 - 高** | 主要功能异常、影响业务流程 | 48 小时 |
| **P2 - 中** | 次要功能异常、有替代方案 | 1 周 |
| **P3 - 低** | UI 问题、体验优化 | 下个版本 |
