## 附录

### A. Bug 严重程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| P0 | 致命 Bug，系统崩溃、数据丢失、核心功能完全不可用 | 应用无法启动、数据库连接失败 |
| P1 | 严重 Bug，主要功能缺陷、数据错误 | 价格计算错误、数据保存失败 |
| P2 | 一般 Bug，次要功能缺陷、UI 问题 | 按钮样式错误、提示信息不准确 |
| P3 | 轻微 Bug，建议改进、拼写错误 | 错别字、颜色不一致 |

### B. Bug 状态定义

| 状态 | 定义 |
|------|------|
| 新建 | Bug 刚被提交，未分配 |
| 进行中 | 开发人员正在修复 |
| 已修复 | 开发人员已修复，待验证 |
| 已验证 | 测试人员验证通过 |
| 已关闭 | Bug 生命周期结束 |
| 重新打开 | 验证未通过，重新修复 |
| 不修复 | 决定不修复（需说明原因） |
