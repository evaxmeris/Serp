## 1. 版本概述

### 1.1 版本背景

Sprint 7 的核心目标是 **完成认证系统重构**，解决原 NextAuth Beta 版本与 Next.js 16 的兼容性问题，采用简化的 JWT 认证方案，确保系统可用。

### 1.2 主要变更

| 变更类型 | 内容 |
|---------|------|
| 新增 | 全新简化 JWT 认证系统 (基于 `jose`) |
| 新增 | 登出 API `/api/auth/logout` |
| 新增 | 登录速率限制 (5 次失败 / 15 分钟) |
| 重构 | 替换不兼容的 NextAuth v5 Beta |
| 安全修复 | 强制检查 `NEXTAUTH_SECRET` 环境变量 |

### 1.3 涉及模块

- 认证模块 (Authentication)
- 权限控制 (Authorization)
- 安全防护 (Security)

## 2. 测试范围

### 2.1 新增功能测试

✅ **必须测试**:

1. **简化 JWT 认证流程**
   - 用户登录成功流程
   - Cookie 设置验证 (HttpOnly, SameSite, maxAge, Secure)
   - JWT 签名和验证
   - 获取当前用户信息 `/api/auth/me`
   - 用户登出流程 `/api/auth/logout`

2. **认证失败场景**
   - 密码错误
   - 用户不存在
   - 账户状态异常 (PENDING_APPROVAL / SUSPENDED / DISABLED)

3. **速率限制功能**
   - 连续 5 次失败不锁定
   - 第 6 次失败触发 429 错误
   - 返回 Retry-After 头
   - 过期后自动重置

4. **权限验证集成**
   - `requireAuth` 中间件正确识别已认证用户
   - `requireAuth` 正确拒绝未认证用户
   - 角色权限验证正常工作 (`ADMIN` > `MANAGER` > `USER` > `VIEWER`)

5. **安全特性验证**
   - 强制检查 `NEXTAUTH_SECRET` 环境变量
   - 密码 bcrypt 加密存储
   - Cookie 安全属性正确

### 2.2 回归测试范围

✅ **必须回归**:

1. **现有 API 认证**
   - 所有 25 个 v1 API 都能正确进行认证
   - 认证通过后正常访问资源
   - 认证失败返回 401

2. **之前已完成模块**
   - 产品管理模块

3. **前端页面**
   - 登录页面正常显示
   - 注册页面正常显示
   - 登录后导航栏正确显示用户名
   - 登出后跳转到登录页

## 7. 风险评估

### 7.1 已识别风险

| 风险 | 可能性 | 影响 | 应对措施 | 状态 |
|------|--------|------|----------|------|
| Cookie Secure 属性在开发环境不匹配 | 低 | 低 | 已根据 `NODE_ENV` 自动判断 | ✅ 已 mitigated |
| 速率限制内存存储重启丢失 | 低 | 低 | 开发环境可接受，生产后续升级 Redis | ⚠️ 接受 |
| 前端页面刷新后状态不同步 | 中 | 中 | 需要手动验证 | 🔍 待测试 |
| 与现有 API 认证中间件不兼容 | 低 | 高 | 所有 API 需要回归测试 | ✅ 计划中 |
| JWT 密钥长度不足 | 中 | 高 | 文档说明要求 ≥32 字符，部署时检查 | 📋 待检查 |

### 7.2 缺陷严重度定义

| 严重度 | 定义 | 示例 |
|--------|------|------|
| Blocker | 系统无法启动，核心功能无法使用 | 环境变量检查不通过无法启动、登录完全失败 |
| Critical | 核心功能不可用，安全问题 | 密码不加密存储、可以绕过认证、权限验证失效 |
| Major | 主要功能失效，业务流程中断 | 登出不成功、速率限制不工作、刷新页面需要重新登录 |
| Minor | 次要功能失效，不影响核心流程 | 错误提示信息不准确、UI 显示问题 |
| Trivial | 细节问题，不影响使用 | 拼写错误、注释问题 |
