## 🔴 P0 - 严重 Bug

### BUG-001: 订单创建 API 返回 500 错误

| 属性 | 值 |
|------|-----|
| **用例 ID** | OM-017 |
| **模块** | 订单管理 |
| **API 端点** | POST /api/orders |
| **发现日期** | 2026-03-06 |
| **发现方式** | 自动化测试 |

**复现步骤:**
```bash
curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "xxx",
    "currency": "USD",
    "paymentTerms": "T/T",
    "deliveryTerms": "FOB",
    "deliveryDate": "2026-03-20",
    "notes": "测试订单",
    "items": [{
      "productName": "测试产品",
      "specification": "标准规格",
      "quantity": 100,
      "unitPrice": "10.00",
      "amount": "1000.00"
    }]
  }'
```

**预期结果:**
- 返回状态码：201
- 返回数据包含订单 ID 和订单号

**实际结果:**
- 返回状态码：500
- 返回数据：`{"error":"Failed to create order"}`

**可能原因:**
1. 后端订单创建逻辑存在异常
2. 订单项 (OrderItem) 创建失败
3. 数据库事务处理问题
4. 缺少必要的字段验证

**建议修复:**
1. 检查服务器日志获取详细错误堆栈
2. 验证订单和订单项的创建逻辑
3. 添加适当的错误处理和日志记录

**状态:** 🔴 待修复  
**指派给:** 开发工程师  
**目标修复版本:** v0.4.0

### BUG-002: 订单金额计算返回 0

| 属性 | 值 |
|------|-----|
| **用例 ID** | OM-003 |
| **模块** | 订单管理 |
| **API 端点** | POST /api/orders |
| **发现日期** | 2026-03-06 |
| **发现方式** | 自动化测试 |

**复现步骤:**
```javascript
const orderData = {
  customerId: "xxx",
  items: [
    { productName: '商品 A', quantity: 10, unitPrice: '5.00', amount: '50.00' },
    { productName: '商品 B', quantity: 20, unitPrice: '3.50', amount: '70.00' },
    { productName: '商品 C', quantity: 15, unitPrice: '8.00', amount: '120.00' }
  ]
};
// POST /api/orders
```

**预期结果:**
- 订单总金额 = 50 + 70 + 120 = 240.00

**实际结果:**
- 订单总金额 = 0

**可能原因:**
1. 后端未正确计算订单项金额总和
2. 金额字段未正确映射到数据库
3. 计算逻辑缺失或返回默认值 0

**建议修复:**
1. 检查订单创建时的金额计算逻辑
2. 验证订单项金额的累加计算
3. 添加金额计算的单元测试

**状态:** 🔴 待修复  
**指派给:** 开发工程师  
**目标修复版本:** v0.4.0

## 🟡 P2 - 轻微 Bug

### BUG-004: 订单编辑限制测试失败（依赖性问题）

| 属性 | 值 |
|------|-----|
| **用例 ID** | OM-005 |
| **模块** | 订单管理 |
| **API 端点** | PUT /api/orders/:id |
| **发现日期** | 2026-03-06 |
| **发现方式** | 自动化测试 |

**问题描述:**
测试用例 OM-005 失败是因为依赖的 testOrder 变量为 null。testOrder 为 null 是因为 OM-017（订单创建测试）失败。

**预期结果:**
- 测试用例正常执行，验证已发货订单不能被编辑

**实际结果:**
- TypeError: Cannot read properties of null (reading 'id')

**可能原因:**
- 测试用例依赖顺序问题
- OM-017 失败导致 testOrder 未赋值

**建议修复:**
- 此问题为测试用例依赖性问题，非功能 Bug
- 待 BUG-001 修复后，此测试用例将自动通过

**状态:** 🟡 依赖 BUG-001 修复  
**指派给:** 测试工程师  
**目标修复版本:** v0.4.0
