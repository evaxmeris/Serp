## 🚀 部署步骤

### 1. 准备阶段

```bash
# 1.1 切换到主分支
git checkout main

# 1.2 拉取最新代码
git pull origin main

# 1.3 检查当前版本
cat package.json | grep version
```

### 2. 数据库迁移

```bash
# 2.1 备份数据库（重要！）
pg_dump -h localhost -U trade_erp trade_erp > backup_$(date +%Y%m%d_%H%M%S).sql

# 2.2 执行数据库迁移
cd trade-erp
npx prisma migrate deploy

# 2.3 验证迁移
npx prisma db pull
npx prisma generate
```

### 3. 构建应用

```bash
# 3.1 安装依赖
npm install

# 3.2 构建应用
npm run build

# 3.3 验证构建
# 检查输出是否有错误
```

### 4. 部署更新

#### Docker 部署（推荐）

```bash
# 4.1 停止旧容器
docker-compose down

# 4.2 构建新镜像
docker-compose build --no-cache

# 4.3 启动新容器
docker-compose up -d

# 4.4 查看容器状态
docker-compose ps

# 4.5 查看日志
docker-compose logs -f
```

#### 直接部署

```bash
# 4.1 重启服务
pm2 restart trade-erp

# 或
systemctl restart trade-erp
```

### 5. 验证部署

```bash
# 5.1 健康检查
curl http://localhost:3000/api/health

# 5.2 版本检查
curl http://localhost:3000/api/version

# 5.3 功能测试
# 访问出库单管理页面
# 创建测试出库单
# 确认功能正常
```

## 🔄 回滚方案

### 回滚条件
- 严重 Bug 影响核心功能
- 数据库迁移失败
- 性能严重下降
- 用户反馈重大问题

### 回滚步骤

```bash
# 1. 停止当前服务
docker-compose down

# 2. 恢复数据库

# 3. 切换代码版本
git checkout <previous-tag>

# 4. 重新构建部署
```
