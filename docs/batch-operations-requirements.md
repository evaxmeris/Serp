## 1. 概述

### 1.1 项目背景

Trade ERP 作为外贸企业资源管理系统，用户日常操作中经常需要对多条数据进行相同处理。逐个操作效率低下，影响用户体验和工作效率。因此需要批量操作功能来提升工作效率。

### 1.2 目标

- 减少重复劳动，提升用户操作效率
- 支持常见批量操作场景
- 提供操作确认和错误处理机制
- 保持界面简洁易用

## 2. 需求分析与优先级

### 2.1 需求优先级评定标准

| 优先级 | 标准 | 权重 |
|--------|------|------|
| P0 | 核心业务流程必需，高频使用 | 必须做 |
| P1 | 重要功能，提升效率，经常使用 | 应该做 |
| P2 |  nice to have，低频使用 | 可以做 |

### 2.2 各模块需求优先级汇总

| 模块 | 功能 | 优先级 | 业务频率 |
|------|------|--------|----------|
| **产品管理** | 批量导入 | P0 | 高（新品上架） |
| **产品管理** | 批量导出 | P0 | 高（数据分析） |
| **产品管理** | 批量删除 | P1 | 中（清理数据） |
| **订单管理** | 批量确认 | P0 | 高（每日订单处理） |
| **订单管理** | 批量发货 | P0 | 高（每日发货） |
| **客户管理** | 批量导入 | P0 | 中（客户迁移） |
| **客户管理** | 批量打标签 | P1 | 中（客户分类） |
| **供应商管理** | 批量导入 | P1 | 中（供应商建档） |
| **供应商管理** | 批量评估 | P2 | 低（季度评估） |

### 2.3 优先级排序

**P0（必须第一阶段开发）:**
1. 产品管理 - 批量导入/导出
2. 订单管理 - 批量确认/发货
3. 客户管理 - 批量导入

**P1（第二阶段开发）:**
1. 产品管理 - 批量删除
2. 客户管理 - 批量打标签
3. 供应商管理 - 批量导入

**P2（第三阶段开发）:**
1. 供应商管理 - 批量评估

## 3. 模块详细需求

### 3.1 产品管理模块

#### 3.1.1 批量导入（P0）

**需求描述:**
- 用户可以通过 Excel/CSV 文件批量导入产品信息
- 支持新建产品和更新现有产品
- 导入前预览数据，显示成功/失败记录
- 失败记录提供错误原因，支持修正后重新导入

**支持的字段:**
- SKU (必填)
- 产品名称 (必填)
- 分类
- 成本价格
- 销售价格
- 最小起订量
- 供应商
- 重量
- 尺寸

**业务规则:**
- SKU 存在时选择：跳过/更新
- 必填字段缺失时标记为错误，整行不导入
- 支持图片 URL 批量导入

**验收标准:**
- ✅ 支持 Excel (.xlsx) 和 CSV 格式
- ✅ 导入前预览，可勾选需要导入的行
- ✅ 显示导入结果统计（成功/失败/跳过）
- ✅ 错误行可下载错误报告，修正后重新导入

#### 3.1.2 批量导出（P0）

**需求描述:**
- 用户可以导出产品列表到 Excel 文件
- 支持导出选中产品或当前筛选条件下的所有产品
- 用户可以选择需要导出的字段

**支持导出字段:**
- 全部字段可选
- 默认勾选常用字段：SKU、名称、分类、价格、供应商

**验收标准:**
- ✅ 支持选择导出范围（选中/全部筛选结果）
- ✅ 支持自定义导出字段
- ✅ 导出格式为 .xlsx，格式规范

#### 3.1.3 批量删除（P1）

**需求描述:**
- 用户可以勾选多个产品进行批量删除
- 删除前需要二次确认
- 支持级联删除选项（是否同时删除相关库存/SKU 记录）

**业务规则:**
- 已有关联订单的产品不允许删除，提示用户
- 删除需要管理员权限或产品编辑权限
- 操作记入操作日志

**验收标准:**
- ✅ 勾选多条后显示"已选择 X 项"
- ✅ 删除前确认弹窗显示明细汇总
- ✅ 不满足删除条件的项明确提示
- ✅ 删除后刷新列表

### 3.2 订单管理模块

#### 3.2.1 批量确认（P0）

**需求描述:**
- 用户可以批量确认待确认订单
- 批量确认后订单状态变为"已确认"
- 支持批量设置确认日期

**前置条件:**
- 订单状态必须是 PENDING（待确认）
- 当前用户有订单确认权限

**后置操作:**
- 更新订单状态为 CONFIRMED
- 记录确认时间和确认人
- 触发订单确认后续工作流（如通知生产/采购）

**验收标准:**
- ✅ 只允许勾选状态为待确认的订单
- ✅ 批量确认后显示成功/失败统计
- ✅ 确认失败的订单显示原因

#### 3.2.2 批量发货（P0）

**需求描述:**
- 用户可以批量对已确认订单进行发货操作
- 支持批量填写物流信息
- 支持批量发送发货通知给客户

**必填信息:**
- 物流服务商（可批量统一设置）
- 物流跟踪号（支持每行不同）

**业务规则:**
- 只有已确认且未发货的订单才能发货
- 发货后订单状态变为 SHIPPED
- 记录发货时间和发货人
- 可选择是否发送邮件/站内通知给客户

**导入方式:**
- 方式一：在列表勾选订单后，在弹窗中填写统一物流信息
- 方式二：通过 CSV 批量导入跟踪号

**验收标准:**
- ✅ 支持勾选多个订单批量发货
- ✅ 支持 CSV 导入跟踪号
- ✅ 发货成功后更新状态和物流信息
- ✅ 可选择是否通知客户

#### 3.3.2 批量打标签（P1）

**需求描述:**

**操作方式:**
- 勾选客户 → 点击"批量标签" → 选择添加/移除 → 选择标签 → 确认

**验收标准:**

## 4. API 设计方案

### 4.1 通用批量操作响应格式

```typescript
// 通用批量请求
interface BatchRequest<T> {
  ids: string[];           // 选中的 ID 列表
  data: Partial<T>;        // 批量更新的数据（对所有选中项应用）
  options?: {
    skipErrors?: boolean;  // 遇到错误是否继续
    dryRun?: boolean;      // 试运行，不实际修改
  };
}

// 通用批量响应
interface BatchResult {
  total: number;           // 总操作数
  success: number;         // 成功数
  failed: number;          // 失败数
  errors: BatchError[];    // 错误详情
  results: BatchItemResult[]; // 每项结果
}

interface BatchError {
  id: string;              // 出错记录 ID
  message: string;         // 错误信息
  code: string;            // 错误码
}

interface BatchItemResult {
  id: string;
  success: boolean;
  message?: string;
}
```

### 4.2 产品管理 API

#### 4.2.1 批量导入

```
POST /api/products/batch/import
Content-Type: multipart/form-data

Parameters:
- file: File (Excel/CSV)
- updateExisting: boolean (是否更新已存在 SKU)
- sheet?: string (Excel 工作表名，默认 Sheet1)

Response:
{
  ...BatchResult,
  preview: ProductImportItem[]  // 预览数据
}
```

#### 4.2.2 批量导出

```
POST /api/products/batch/export
Body:
{
  ids?: string[]               // 选中的 ID，不填则使用筛选条件
  filters?: FilterParams       // 当前筛选条件
  fields: string[]             // 要导出的字段
}

Response: 下载 Excel 文件
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

#### 4.2.3 批量删除

```
POST /api/products/batch/delete
Body:
{
  ids: string[]
  cascade: boolean  // 是否级联删除关联数据
}

Response: BatchResult
```

### 4.3 订单管理 API

#### 4.3.1 批量确认

```
POST /api/orders/batch/confirm
Body:
{
  ids: string[]
  confirmedAt?: DateTime  // 默认当前时间
}

Response: BatchResult
```

#### 4.3.2 批量发货

```
POST /api/orders/batch/ship
Body:
{
  // 方式一：统一物流信息
  ids: string[]
  carrier: string
  trackingNumbers?: {[orderId: string]: string} // 每个订单不同跟踪号
  sendNotification: boolean  // 是否通知客户
  
  // 方式二：文件上传导入跟踪号
  // 使用 multipart/form-data
}

Response: BatchResult
```

### 4.4 客户管理 API

#### 4.4.1 批量导入

```
POST /api/customers/batch/import
Content-Type: multipart/form-data

Parameters:
- file: File
- updateExisting: boolean
- assignSalesRepId?: string

Response: BatchResult
```

#### 4.4.2 批量标签操作

```
POST /api/customers/batch/tags
Body:
{
  ids: string[]
  action: 'add' | 'remove'
  tags: string[]
}

Response: BatchResult
```

### 4.5 供应商管理 API

#### 4.5.1 批量导入

```
POST /api/suppliers/batch/import
Content-Type: multipart/form-data

Parameters:
- file: File
- updateExisting: boolean

Response: BatchResult
```

#### 4.5.2 批量评估

```
POST /api/suppliers/batch/evaluate
Body:
{
  ids: string[]
  rating?: 'A' | 'B' | 'C' | 'D'
  status?: 'ACTIVE' | 'INACTIVE'
  notes?: string
}

Response: BatchResult
```

### 4.6 导入模板下载 API

为方便用户准备导入文件，提供模板下载：

```
GET /api/:module/batch/template
Response: 下载 Excel 模板文件
```

支持:
- `/api/products/batch/template`
- `/api/customers/batch/template`
- `/api/suppliers/batch/template`

## 5. UI 设计方案

### 5.1 列表批量操作通用交互模式

#### 5.1.1 批量选择设计

**布局:**
- 列表第一列添加复选框
- 表头有"全选/取消全选"复选框
- 底部悬浮批量操作栏，显示"已选择 X 项"
- 批量操作栏滑出动画，选择后显示

**状态:**
- 未选择任何项：批量操作栏隐藏
- 已选择：显示操作栏，按钮根据用户权限启用/禁用

```
┌─────────────────────────────────────────────────────────────┐
│ [ ] SKU  产品名称  价格  供应商  状态  操作                  │
├─────────────────────────────────────────────────────────────┤

│ [ ] SKU003  Product C  $20.00  Supplier C  Active  [...]   │
└─────────────────────────────────────────────────────────────┘

│ [ 导入 ] [ 导出 ] [ 删除 ]                         │
└──────────────────────────────────────────────────┘
```

#### 5.1.2 分步式弹窗流程

对于复杂批量操作（如导入、发货），采用分步弹窗向导：

**Step 1: 上传文件 / 输入信息**
- 文件上传区域支持拖拽
- 模板下载链接
- 选项设置（是否更新已存在）

**Step 2: 预览并确认**
- 表格预览待导入数据
- 每行标记：新增/更新/错误
- 错误行标注原因，可直接编辑
- 用户可取消勾选不导入错误行

**Step 3: 执行并显示结果**
- 进度条显示导入进度
- 结果统计：成功 X / 失败 Y / 跳过 Z
- 可下载错误报告
- 关闭弹窗后刷新列表

## 6. 技术实现考虑

### 6.1 前端考虑

1. **文件处理**
   - 使用 `xlsx` 库解析 Excel 文件
   - 前端预览减少服务器压力
   - 大文件分片上传支持

2. **状态管理**
   - 批量操作进行时锁定 UI
   - 进度实时更新

3. **性能考虑**
   - 支持大量数据（千条级别）
   - 分页列表中跨页选择支持

### 6.2 后端考虑

1. **事务处理**
   - 可选择全成功才提交/部分成功部分提交
   - 默认配置：遇到错误继续，记录错误

2. **并发控制**
   - 批量操作使用队列，避免超时
   - 支持异步处理，轮询进度

3. **权限校验**
   - 每条记录都要校验权限
   - 防止越权操作

4. **日志记录**
   - 所有批量操作记入操作日志
   - 支持审计追溯

### 6.3 导入模板规范

**示例：产品导入模板格式**
