## 🔄 协作机制

### 任务流转

```
需求 → 需求分析师 → 需求文档 → 架构师 → 技术设计 → 开发工程师 → 代码
                                                              │
                                                              ▼
发布 ← 项目经理 ← 验收 ← 测试经理 ← 测试报告 ← 测试子代理 ← CI/CD
```

### 沟通机制

- **每日站会** - 各子代理报告进度和阻塞
- **Sprint 评审** - 每期结束展示成果
- **需求评审** - 需求文档完成后评审
- **技术评审** - 技术方案完成后评审

### 文档管理

| 文档类型 | 负责人 | 存储位置 |
|----------|--------|----------|
| 需求文档 | 需求分析师 | `docs/requirements/` |
| 技术设计 | 架构师 | `docs/architecture/` |
| 测试报告 | 测试经理 | `docs/testing/` |
| 项目计划 | 项目经理 | `docs/project/` |
| API 文档 | 架构师 | `docs/api/` |
