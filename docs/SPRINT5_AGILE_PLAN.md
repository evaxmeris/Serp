## 🎯 阶段转换规则

### 完成标准（Definition of Done）
每个阶段必须满足以下条件才能标记为 DONE：

1. **功能完整**: 所有计划任务完成
2. **测试通过**: 相关测试用例 100% 通过
3. **代码审查**: 至少 1 人审查通过
4. **文档更新**: 相关文档已更新
5. **Git 提交**: 代码已提交并推送

### 阶段转换流程
```
IN PROGRESS → 自检完成 → 代码审查 → 测试验证 → DONE
```
