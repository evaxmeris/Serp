## 🎯 **可选认证方案**

### 方案 A: 简化 JWT 认证 (推荐 ⭐⭐⭐⭐⭐)

**技术栈**:
```typescript
// 核心依赖
{
  "jose": "^5.x",      // JWT 生成和验证
  "bcryptjs": "^3.x"   // 密码加密 (已有)
}
```

**实现方式**:
```typescript
// 1. 登录时生成 JWT
const token = await new SignJWT({ id, email, role })
  .setProtectedHeader({ alg: 'HS256' })
  .setExpirationTime('7d')
  .sign(SECRET);

// 2. 设置 HttpOnly cookie
cookies().set('auth-token', token, {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production',
  sameSite: 'lax',
  maxAge: 7 * 24 * 60 * 60,
  path: '/',
});

// 3. API 验证 token
const token = cookies().get('auth-token')?.value;
const { payload } = await jwtVerify(token, SECRET);
```

**优点**:
- ✅ 代码简单，易于理解
- ✅ 完全可控，无黑盒
- ✅ 无外部服务依赖
- ✅ 适合中小型项目
- ✅ 已有 `auth-simple.ts` 实现基础
- ✅ 适合 Next.js 16 + React 19

**缺点**:
- ❌ 需要自行实现所有逻辑
- ❌ 无社区生态支持
- ❌ 第三方登录需手动实现

**工作量**: 2-3 天

**实施步骤**:
1. 安装 `jose` 库
2. 修改登录 API 调用 `auth-simple.ts`
3. 更新 `/api/auth/me` 使用 JWT 验证

### 第 6 步：添加速率限制 (2 小时)

**文件**: `src/lib/rate-limit.ts` (新建)

```typescript
import { NextRequest, NextResponse } from 'next/server';

const limitMap = new Map<string, { count: number; resetTime: number }>();

export function rateLimit(
  request: NextRequest,
  options: {
    limit: number;      // 最大请求次数
    windowMs: number;   // 时间窗口（毫秒）
  }
): NextResponse | null {
  const ip = request.headers.get('x-forwarded-for') || 'unknown';
  const now = Date.now();
  
  const record = limitMap.get(ip) || { count: 0, resetTime: now + options.windowMs };
  
  if (now > record.resetTime) {
    record.count = 1;
    record.resetTime = now + options.windowMs;
  } else {
    record.count += 1;
  }
  
  limitMap.set(ip, record);
  
  if (record.count > options.limit) {
    return NextResponse.json(
      { 
        error: 'Too many requests',
        retryAfter: Math.ceil((record.resetTime - now) / 1000)
      },
      { status: 429 }
    );
  }
  
  return null;
}
```

**在登录 API 中使用**:

  });

    return limitError;
  }
