# Trade ERP v0.8.0 - 导航结构设计

## 导航系统概述

Trade ERP 采用**左侧可收缩导航栏 + 顶部面包屑/操作栏**的双层导航结构，支持多终端响应式适配。

### 核心设计原则

1. **业务模块化组织** - 按跨境电商业务流程自然分组
2. **可收缩交互** - 大屏展开全菜单，小屏自动折叠
3. **视觉层次清晰** - 一级菜单突出，二级菜单缩进
4. **图标化识别** - 每个菜单项配专属图标，增强识别性

## 导航栏布局设计

### 展开状态（桌面端）

| 区域 | 宽度 | 说明 |
|------|------|------|
| 导航栏 | 240px | 固定左侧，显示图标 + 文字 |
| 内容区 | 自适应 | 占满剩余空间 |

### 折叠状态（平板/移动端）

| 区域 | 宽度 | 说明 |
|------|------|------|
| 导航栏 | 64px | 仅显示图标，悬浮显示文字提示 |
| 内容区 | 自适应 | 占满剩余空间 |

### 交互行为

1. **折叠/展开按钮** - 导航栏顶部点击 ≡ 图标切换
   - 展开状态：显示图标 + 文字
   - 折叠状态：仅显示图标，hover 显示 tooltip
   - 移动端：自动变为抽屉式，从左侧滑出

2. **菜单分组**
   - 每个业务模块之间有 8px 间距分隔
   - 常用模块（订单、商品）放置在上方
   - 低频模块（设置、报表）放置在底部

3. **激活状态**
   - 当前页面菜单项高亮显示（背景色 + 主色调左侧边）
   - 父菜单包含当前子菜单时，父菜单也保持高亮

## 响应式导航规则

| 断点 | 导航行为 |
|------|----------|
| ≥1200px (桌面) | 默认展开，可手动折叠 |
| 768px - 1199px (平板) | 默认折叠，可点击展开 |
| <768px (手机) | 汉堡按钮点击后，从左侧滑出抽屉（遮罩层） |

### 抽屉交互（移动端）

- 点击汉堡按钮 → 抽屉滑入 + 背景遮罩显示
- 点击遮罩或返回 → 抽屉滑出
- 抽屉宽度：屏幕宽度 × 0.8
- 高度：全屏

# 新增页面详细设计

## 1. 角色权限管理页面（/settings/roles）

### 页面布局

```
┌─────────────────────────────────────────────────┐
│ 面包屑: 系统设置 > 角色权限  [+ 新建角色]         │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│ 角色列表     │ 角色详情 / 权限配置              │
│ (侧边栏)     │                                  │
│  ┌───────┐   │  基本信息                        │
│  │ 管理员│   │  - 角色名称                      │
│  └───────┘   │  - 角色描述                     │
│  ┌───────┐   │  - 是否启用                     │
│  │ 运营  │   │                                  │
│  └───────┘   │  权限分配 (树形结构)            │
│  ┌───────┐   │  ┌─────────────────────────┐   │
│  │ 客服  │   │  │ □ 订单管理              │   │
│  └───────┘   │  │   ■ 查看订单            │   │
│  ┌───────┐   │  │   ■ 创建订单            │   │
│  │ 财务  │   │  │ □ 商品管理              │   │
│  └───────┘   │  │ ...                     │   │
│              │  └─────────────────────────┘   │
│              │                                  │
│              │       [保存] [取消]             │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

### 布局结构

| 区域 | 宽度占比 | 说明 |
|------|----------|------|
| 角色列表侧边栏 | 24% | 固定宽度 280px，显示所有角色，支持搜索过滤 |
| 权限配置区 | 76% | 自适应剩余宽度，显示选中角色的详细权限配置 |

### 功能模块

#### 左侧：角色列表
- 搜索框：按角色名称搜索
- 角色列表项：显示角色名称、用户数量、状态（启用/禁用）
- 新建角色按钮：固定在列表顶部
- 点击角色项切换右侧内容，当前选中高亮

#### 右侧：权限配置
- **基本信息区域**
  - 角色名称输入框
  - 角色描述文本框
  - 启用/禁用开关
  - 创建时间、更新时间（只读）

- **权限分配区域**
  - 按模块树形展示所有权限
  - 支持全选/反选模块
  - 级联选择：勾选父模块自动勾选所有子权限
  - 权限节点类型：
    - 页面访问权限
    - 操作权限（新建/编辑/删除/导出）
    - 数据范围权限（全部/本部门/仅自己）

### 交互流程

```
进入页面
  ↓
默认选中第一个角色，显示其权限
  ↓
[操作选项]

1. 搜索角色 → 过滤列表 → 点击目标角色 → 显示权限
   
2. 点击「新建角色」
   ↓
清空表单 → 输入基本信息 → 勾选权限 → 点击保存
   ↓
保存成功 → 刷新角色列表 → 选中新建角色

3. 修改现有角色
   ↓
修改信息或权限 → 点击保存
   ↓
验证权限配置 → 保存成功 → 更新列表

4. 删除角色
   ↓
二次确认弹窗 → 确认删除 → 删除成功 → 刷新列表
   (不能删除内置管理员角色，不能删除有关联用户的角色)
```

### 特殊约束
- 内置「超级管理员」角色不允许删除和修改权限
- 删除角色前必须检查是否有用户关联，有则提示先移除用户
- 权限配置支持批量快速复制（从已有角色复制权限模板）

- 关键词搜索：搜索订单号/买家信息/商品名称

#### 批量操作
- 批量同步到本地
- 批量标记发货
- 批量导出 Excel

#### 快捷操作菜单
- 查看详情：侧滑弹窗显示订单详情

- 去发货：跳转到发货管理页面

```

  ↓
[单订单操作]

[批量操作]

  → 显示处理结果报告: 成功 N 个, 失败 M 个

[导出]

```

### 特殊设计

- 异常标记：同步失败的订单标红，支持一键重新同步
- 悬浮预览：鼠标悬浮订单号显示订单商品缩略图

## 5. 仓库管理页面（/warehouses）

### 页面设计说明

管理多个实体仓库（国内仓、海外仓），支持仓库信息配置、库存仓位管理等。

### 页面布局

```
┌─────────────────────────────────────────────────┐
│ 面包屑: 库存管理 > 仓库管理  [+ 新建仓库]        │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│ 仓库列表     │ 仓库详情 / 配置                   │
│ (侧边栏)     │                                  │
│  ┌───────┐   │  基本信息                        │
│  │ 国内仓│   │  - 仓库名称                      │
│  └───────┘   │  - 仓库编码                     │
│  ┌───────┐   │  - 仓库地址                     │
│  │美西仓│   │  - 联系人 / 电话                │
│  └───────┘   │  - 状态: 启用 / 禁用           │
│  ┌───────┐   │                                  │
│  │英南仓│   │  仓位管理                        │
│  └───────┘   │  表格: 仓位编码 | 位置 | 容量   │
│              │  [添加仓位] [删除]              │
│              │                                  │
│              │  统计信息                        │
│              │  - SKU 总数: N                  │
│              │  - 总库存: X 件                │
│              │  - 总货值: Y                   │
│              │                                  │
│              │        [保存] [删除]            │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

### 布局结构

| 区域 | 宽度 | 说明 |
|------|------|------|
| 仓库列表侧边栏 | 260px | 固定宽度，显示所有仓库 |
| 仓库详情区 | 自适应 | 显示选中仓库的详细配置 |

### 功能模块

#### 左侧：仓库列表
- 搜索过滤：按仓库名称/编码搜索
- 仓库列表项：显示仓库名称、仓库类型、状态
- 类型颜色标识：
  - 国内仓：蓝色
  - 海外仓：紫色
  - 中转仓：橙色
- 点击切换仓库，当前选中高亮

#### 右侧：仓库详情配置

- **基本信息区域**
  - 仓库名称
  - 仓库编码（唯一标识）
  - 仓库类型：下拉选择（国内仓/海外仓/中转仓）
  - 仓库地址：详细地址
  - 联系人、电话、邮箱
  仓库状态：启用/禁用开关

- **仓位管理区域**
  - 表格展示现有仓位（可选，精细化管理使用）
  - 仓位信息：仓位编码、位置描述、最大容量、已用容量
  - 支持添加/删除仓位
  - 如果不需要精细化管理，可以不设置仓位

- **统计信息区域**（只读）
  - 当前仓库 SKU 数量
  - 总库存件数
  - 总库存货值
  - 最近入库时间
  - 最近出库时间

### 交互流程

```
进入页面
  ↓
默认选中第一个仓库，显示详情
  ↓
[操作选项]

1. 新建仓库
   ↓
点击「新建仓库」→ 清空表单 → 填写基本信息 → （可选）添加仓位 → 保存
   ↓
保存成功 → 刷新侧边栏列表 → 选中新建仓库

2. 编辑仓库
   ↓
修改信息或仓位 → 点击保存 → 保存成功

3. 删除仓库
   ↓
检查仓库是否还有库存 → 如果有库存 → 提示无法删除
   ↓
无库存 → 二次确认 → 删除成功 → 刷新列表

4. 禁用仓库
   ↓
切换开关禁用 → 禁用后不允许新入库，但可以出库
   ↓
不影响现有库存数据
```

### 业务规则

- 至少要有一个启用的仓库（默认仓库无法删除）
- 有库存的仓库不允许删除，只允许禁用
- 每个订单的收货/发货都必须关联仓库
- 出库时优先从默认仓库扣减库存，可手动选择

## 设计总结

### 导航结构调整要点

1. **系统设置**新增两个管理页面：角色权限、平台账号，完善了系统管理功能
2. **订单管理**新增平台订单列表，支持跨平台订单统一查看
3. **商品管理**新增产品刊登，实现一键多平台发布
4. **库存管理**新增独立的仓库管理页面，支持多仓精细化管理

### 设计原则保持一致

- 保持左侧可收缩导航架构不变
- 保持图标化设计语言不变
- 保持响应式断点不变
- 保持交互模式一致，用户学习成本低

### 新增功能满足业务需求

- 角色权限：支持细粒度权限控制，适合团队协作
- 平台账号：集中管理多平台授权，统一配置入口
- 平台订单：跨平台订单聚合，提升管理效率
- 产品刊登：一键多平台发布，节省运营时间
- 仓库管理：支持多仓运营，满足跨境业务需求
