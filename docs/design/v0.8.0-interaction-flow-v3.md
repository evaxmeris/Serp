
## 2. 审批流程交互（六类审批）

### 2.1 六类审批清单

| 审批类型 | 申请人 | 审批人 | 多级审批 |
|----------|--------|--------|----------|
| 1. 报销审批 | 员工 | 部门经理 → 财务 | ✅ 两级 |
| 2. 采购审批 | 采购专员 | 运营经理 → 财务 | ✅ 两级 |
| 3. 绩效提成审批 | 绩效管理员 | 老板 → 财务 | ✅ 两级 |
| 4. 出库审批 | 仓库管理员 | 销售经理 | 一级 |
| 5. 入库审批 | 仓库管理员 | 采购经理 | 一级 |
| 6. 用户注册审批 | 新用户 | 超级管理员 | 一级 |

### 2.2 通用审批交互流程

```
申请人发起审批申请
       │
       ▼
填写申请信息 + 上传附件（可选）
       │
       ▼
提交申请 → 状态 = 审批中
       │
       ▼
通知第一审批人（APP/邮件/系统提醒）
       │
       ▼
审批人处理
       │
       ├─ 通过
       │    │
       │    ▼
       │   是否还有下一级？
       │    ├─ 是 → 通知下一级
       │    └─ 否 → 通过，通知申请人，更新状态
       │
       ├─ 拒绝
       │    │
       │    ▼
       │    填写拒绝原因 → 通知申请人 → 状态 = 已拒绝 → 结束
       │
       └─ 退回修改
            │
            ▼
            退回给申请人 → 申请人修改重新提交
```

## 3. 多平台订单管理流程

### 3.1 订单自动同步流程

```
定时任务（每15分钟）
       │
       ▼
调用各平台API（亚马逊、阿里国际站、独立站等）
       │
       ▼
拉取新订单/订单状态变化
       │
       ▼
比对本地订单号
       │
       ├─ 新订单 → 创建本地订单 → 通知销售
       │
       ▼ 已有订单
更新订单状态（如已发货 → 已签收）
       │
       ▼
记录同步日志
       │
       ▼
完成
```

**用户交互流程：**

```
用户进入订单列表页
       │
       ▼
看到所有平台订单统一列表
       │
       ▼
支持按平台筛选、按状态筛选
       │
       ▼
点击订单查看详情 → 显示平台来源、原始单号
       │
       ▼
用户处理发货 → 系统同步发货信息回平台
       │
       ▼
完成 ✅
```

### 3.2 手动创建订单流程

```
销售业务员点击 [➕ 新建订单]
       │
       ▼
选择客户（新建或选择已有）
       │
       ▼
添加产品 SKU → 自动带出价格、库存
       │
       ▼
填写运费、折扣、备注
       │
       ▼
自动计算总金额
       │
       ▼
预览订单 → [提交]
       │
       ▼
订单状态 = 待付款 / 待发货（根据配置）
       │
       ▼
通知相关人员（仓库、财务）
       │
       ▼
完成 ✅
```

**交互要点：**
- 支持快速添加多个SKU
- 实时计算价格、税费、运费
- 检查库存是否充足，不足提示
- 保存草稿功能

## 6. 利润核算流程 💰

### 6.1 按订单核算利润

这是跨境电商最核心的流程！

```
进入订单详情 → 点击 [利润核算]
       │
       ▼
系统自动带出各项数据：
       │
       ├─ 销售收入 = 产品售价 + 运费 - 优惠
       ├─ 产品成本 = 采购成本 × 数量
       ├─ 物流费用 = 国内运费 + 国际运费
       ├─ 平台佣金 = 销售收入 × 佣金比例
       ├─ 广告费用 = 分摊广告费（手动/自动）
       └─ 其他费用 = 打包费、平台手续费等
       │
       ▼
自动计算：
       │
       净利润 = 销售收入 - （产品成本 + 物流费用 + 平台佣金 + 广告费用 + 其他）
       利润率 = (净利润 ÷ 销售收入) × 100%
       │
       ▼
用户可手动调整各项数据
       │
       ▼
[重新计算] → 实时更新结果
       │
       ▼
确认无误 → [保存]
       │
       ▼
核算结果记入系统 → 用于绩效计算和财务报表
       │
       ▼
完成 ✅
```

### 6.2 按SKU核算利润

```
选择产品SKU → 选择时间范围
       │
       ▼
系统汇总：
       │
       ├─ 总销售数量
       ├─ 总收入
       ├─ 总成本（采购 + 头程 + 配送）
       └─ 总广告费用
       │
       ▼
计算：总利润、单位利润、利润率
       │
       ▼
生成利润分析报表
       │
       ▼
完成 ✅
```

### 6.3 按时间段汇总核算

```
选择开始日期 → 结束日期
       │
       ▼
选择维度：按业务员 / 按产品 / 按平台
       │
       ▼
系统计算：
       │
       ├─ 总订单数
       ├─ 总收入
       ├─ 总成本
       ├─ 总利润
       └─ 平均利润率
       │
       ▼
生成汇总报表 → 可导出Excel
       │
       ▼
完成 ✅
```

### 6.4 交互设计要点

```
利润核算卡片交互：
┌─────────────────────────────────┐
│  销售收入              $1,200.00 │ ← 正数黑色
│  减：产品成本            -$450.00 │ ← 负数红色
│  减：物流费用            -$120.00 │
│  减：平台佣金 3%         -$ 36.00 │
│  减：广告分摊            -$ 60.00 │
│  减：其他费用            -$ 15.00 │
├─────────────────────────────────┤
│  **净利润**           **$519.00** │ ← 加粗突出，如果是负数标红
│  **利润率**           **43.25%** │
└─────────────────────────────────┘
```

- 公式透明化，每一项都可以查看明细
- 支持手动调整，系统保留调整痕迹
- 支持一键重新计算
- 最终结果在订单列表颜色标识（利润率高 → 绿色，低 → 红色）

## 7. 绩效提成计算流程 📈

### 7.1 阶梯提成规则

系统支持**阶梯式提成**，卖得越多，比例越高：

示例规则：
```
销售额阶梯：
• 0 - 10,000 → 提成比例 3%
• 10,001 - 30,000 → 提成比例 5%
• 30,001 以上 → 提成比例 8%
```

计算方式：超额累进（类似个人所得税）

### 7.2 月度提成计算流程

```
进入绩效模块 → 选择月份 → 选择业务员
       │
       ▼
系统自动拉取该业务员月度数据：
       │
       ├─ 总销售额
       ├─ 总净利润
       ├─ 订单数量
       └─ 已收款金额
       │
       ▼
根据绩效规则自动计算：
       │
       1. 按阶梯计算销售提成
       2. 加上利润分成（可选，按净利润百分比）
       3. 减去扣款（罚款、借款等）
       4. = 实发提成
       │
       ▼
显示计算明细，每一步都可查看
       │
       ▼
管理员可手动调整（加减项）
       │
       ▼
点击 [重新计算] → 更新结果
       │
       ▼
确认无误 → [提交审批]
       │
       ▼
老板审批 → 财务审批
       │
       ▼
审批通过 → 财务发放 → 记入工资表
       │
       ▼
业务员可在自己账号看到提成明细
       │
       ▼
完成 ✅
```

### 7.3 计算明细展示

```
┌─────────────────────────────────────────┐
│  销售人员：李雷  月份：2026年3月        │
├─────────────────────────────────────────┤
│  总销售额：               $58,600.00     │
│  总净利润：               $18,200.00     │
│  订单数：                   38 单        │
├─────────────────────────────────────────┤
│  阶梯计算：                              │
│  0-10000   × 3%  =  10000 × 0.03 = $300 │
│  10001-30000 × 5% =  20000 × 0.05 = $1000│
│  30001-58600 × 8% =  28600 × 0.08 = $2288│
├─────────────────────────────────────────┤
│  应发提成：               $3,588.00     │
│  减：扣款（上月借款）        -$200.00     │
├─────────────────────────────────────────┤
│  💰 **实发提成**       💰 **$3,388.00** │
└─────────────────────────────────────────┘
```

### 7.4 不同角色提成方案

**方案1：销售业务员**
- 基础底薪 + 销售额阶梯提成 + 净利润分成

**方案2：采购专员**
- 底薪 + (采购成本节约额) × 比例提成

**方案3：运营经理**
- 底薪 + 团队总利润 × 管理提成

系统支持：
- 每个角色可以配置不同提成规则
- 支持按个人 / 按团队
- 支持阶梯和固定比例两种模式

## 8. 交互设计原则

### 8.1 用户体验原则

1. **角色适配** → 什么角色看到什么功能，不多不少
2. **一步到位** → 常用操作不超过三次点击
3. **实时反馈** → 操作后立即给出反馈（成功/失败）
4. **可撤销** → 删除等操作要有确认提示，支持撤销
5. **透明化** → 计算过程全部可见，不是黑盒
6. **少即是多** → 界面只显示当前需要的信息，不拥挤

### 8.2 错误处理交互

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示重试按钮，保留用户输入 |
| 权限不足 | 清晰提示"您没有此操作权限，请联系管理员" |
| 表单校验失败 | 在错误字段下方显示红色提示，不跳到顶部 |
| 数据加载中 | 显示骨架屏，不白屏 |
| 空数据 | 友好提示 + 引导操作按钮（如"点击新建第一个订单"） |

### 8.3 快捷键支持（桌面端）

- `Ctrl + N` → 新建订单
- `Ctrl + S` → 保存当前表单
- `Ctrl + F` → 搜索
- `Esc` → 关闭模态框/返回
