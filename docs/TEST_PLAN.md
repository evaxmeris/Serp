# 角色权限管理系统测试计划

## 1. 测试策略

### 1.1 测试级别

| 测试级别 | 说明 | 优先级 |
|---------|------|--------|
| 单元测试 | 测试单个权限验证函数、数据访问层 | P1 |
| 集成测试 | 测试 API 接口、数据库操作、前后端交互 | P1 |
| 系统测试 | 测试完整的角色权限管理流程 | P1 |
| 回归测试 | 确保现有核心角色和用户权限不受影响 | P1 |
| 安全测试 | 测试权限越权、注入防护等安全问题 | P2 |
| E2E 测试 | 端到端完整流程测试 | P2 |

### 1.2 测试方法

- **黑盒测试**：从用户角度验证功能正确性
- **白盒测试**：验证权限逻辑和数据访问的正确性
- **静态代码分析**：检查权限验证逻辑是否存在遗漏
- **渗透测试**：模拟攻击者尝试越权访问

### 1.3 退出标准

- 所有 P1 测试用例执行通过
- 无严重缺陷（blocker/critical）
- P2 缺陷不影响核心功能使用
- 回归测试全部通过

## 2. 测试范围

### 2.1 功能测试范围

✅ **包含**：
- 角色 CRUD 操作（创建、读取、更新、删除）
- 权限配置功能（权限树、权限勾选、批量分配）
- 用户角色分配（单个/批量分配、移除）
- 权限验证（菜单访问、按钮权限、数据权限）
- 系统角色保护（不可删除系统内置角色）

❌ **不包含**：
- LDAP/SSO 第三方权限集成
- 数据权限行级过滤的详细业务规则（单独测试）

### 2.2 集成测试范围

- 前后端集成（前端页面 → 后端 API → 数据库）
- 数据库操作（事务一致性、外键约束、唯一约束）
- API 接口（请求验证、权限拦截、响应格式）
- 与用户管理模块集成
- 与菜单模块集成

### 2.3 回归测试范围

- 现有 5 个核心角色（ADMIN、SALES、PURCHASING、WAREHOUSE、VIEWER）
  - 角色数据完整性
  - 权限配置完整性
- 现有用户角色分配关系保持不变
- 现有 API 接口兼容性（不破坏向后兼容性）
- 登录权限验证正常工作

### 2.4 安全测试范围

- 未认证用户访问拦截
- 未授权用户访问拦截
- 水平越权测试（普通用户尝试访问其他用户的角色管理）
- 垂直越权测试（低权限用户尝试访问管理功能）
- SQL 注入防护测试
- XSS 防护测试（角色名称/描述输入）

## 6. 缺陷严重等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| Blocker | 阻塞整个测试或核心功能无法使用 | 无法访问角色管理页面，登录后崩溃 |
| Critical | 核心功能错误，严重影响使用 | 权限验证不生效，可以越权访问 |
| Major | 主要功能错误，影响部分功能使用 | 无法删除角色，可以创建但无法编辑 |
| Minor | 次要功能错误，不影响核心功能 | 分页错误，排序错误，UI 显示问题 |
| Trivial | 轻微问题，文字错误、样式问题 | 错别字，对齐问题 |
