## 数据库设计变更

### 问题修复: 移除冗余表

**问题**: 同时存在 `Inventory` 和 `InventoryItem` 两张表，功能重复。

**解决方案**: 移除 `Inventory`，统一使用 `InventoryItem`。

**变更内容:**

1. ✅ `InventoryItem` 增强: 添加 `minStock`, `maxStock`, `lastInboundDate`, `lastOutboundDate` 字段
2. ✅ `InventoryItem` 添加关系: `logs InventoryLog[]`
3. ✅ `InventoryLog` 更新关系: 移除 `inventoryId`，关联到 `InventoryItem`
4. ✅ `Product` 移除 `inventories Inventory[]`
5. ✅ `Warehouse` 移除 `inventories Inventory[]`
6. ✅ `StockMovement` 添加缺失索引
7. ✅ `InboundOrder` 添加缺失索引
8. ✅ `Customer` 添加缺失索引: `@@index([status, createdAt])`, `@@index([ownerId])`
9. ✅ `Inquiry` 添加缺失索引: `@@index([assignedTo, status])`

### 迁移步骤

```bash
# 1. 备份数据库
pg_dump -U $user $db > backup-v0.6.0-$(date +%Y%m%d).sql

# 2. 生成迁移文件
npx prisma migrate dev --name v0.7.0-remove-redundant-inventory-table

# 3. 执行数据迁移
ts-node scripts/migrate-inventory-to-inventory-item.ts

# 4. 验证数据一致性
#    比较计数: SELECT COUNT(*) FROM inventory_items;
#    抽查库存数量是否正确

# 5. 测试核心功能
#    - 入库操作
#    - 出库操作
#    - 库存查询
#    - 库存变动记录
```

### 回滚方案

```bash
# 1. 恢复备份
psql -U $user $db < backup-v0.6.0-xxxxxx.sql

# 2. 回滚迁移
npx prisma migrate resolve --rolled-back v0.7.0-remove-redundant-inventory-table

# 3. 重启应用
```

## API 接口规范统一

### 版本化规范

- 新模块: `/api/v1/` 前缀
- 旧模块: 保持原路径，逐步迁移
- 当前完整路由清单见 `docs/architecture/v0.7.0-API_SPECIFICATION.md`

### 统一响应格式

所有 API 必须遵守:

```typescript
{
  "success": boolean,
  "code": string,
  "data": T,
  "message": string,
  "timestamp": ISO8601
}
```

### 错误码统一

- 通用错误码 + 业务模块错误码
- 完整错误码清单见规范文档

## 新增功能模块

### 1. 仓储管理模块

**包含:**
- 仓库管理 `Warehouse`
- 入库管理 `InboundOrder`
- 出库管理 `OutboundOrder`
- 库存查询 `InventoryItem`
- 库存变动记录 `InventoryLog`, `StockMovement`

**API 清单:**
```

POST   /api/v1/inbound-orders
GET    /api/v1/inbound-orders/:id
PUT    /api/v1/inbound-orders/:id
DELETE /api/v1/inbound-orders/:id
POST   /api/v1/inbound-orders/:id/confirm
POST   /api/v1/inbound-orders/:id/cancel

GET    /api/v1/outbound-orders
POST   /api/v1/outbound-orders
POST   /api/v1/outbound-orders/batch
GET    /api/v1/outbound-orders/:id
PUT    /api/v1/outbound-orders/:id
DELETE /api/v1/outbound-orders/:id
POST   /api/v1/outbound-orders/:id/confirm
POST   /api/v1/outbound-orders/:id/cancel

GET    /api/v1/inventory
GET    /api/v1/inventory/logs
```

### 2. 财务报表模块

**包含:**
- 报表定义 `ReportDefinition`
- 报表数据 `ReportData`
- 报表订阅 `ReportSubscription`
- 导出日志 `ReportExportLog`
- 定时计划 `ReportSchedule`

**支持报表类型:**
- 销售统计表
- 利润分析表
- 库存统计表
- 采购统计表
- 现金流量表
- 自定义报表

**API 清单:**
```

GET    /api/v1/reports/purchase
GET    /api/v1/reports/cashflow

POST   /api/v1/reports/export
```
