# API 认证测试发现的 Bug 清单

## 已修复的 Bug

### BUG-AUTH-001: JWT 密钥不一致导致认证失败
- **严重程度**: 🔴 严重
- **状态**: ✅ 已修复
- **描述**: `auth-simple.ts` 和 `auth-api.ts` 两个认证模块在模块加载时使用静态常量保存密钥，由于 Next.js 开发服务器的模块缓存机制，导致两个模块拿到的密钥不一致，签名验证永远失败，所有 API 都返回 401。
- **影响范围**: 所有需要认证的 API 接口
- **根因分析**: 
  - 模块级常量在服务启动时初始化
  - 环境变量加载时机与模块初始化时机不一致
  - 模块缓存导致密钥无法动态更新
- **解决方案**: 改为每次使用时动态获取密钥，保证签名和验证使用同一个密钥
- **修复文件**: 
  - `src/lib/auth-simple.ts`
  - `src/lib/auth-api.ts`

## 待处理的问题

### PERF-001: 单个资源查询接口响应时间过长
- **严重程度**: 🟡 中等
- **状态**: ⏳ 待处理
- **描述**: 单个资源查询接口（如 `/api/customers/1`）响应时间约 480ms，超过 200ms 的性能要求
- **影响接口**: 
  - `/api/customers/:id`
  - `/api/v1/inbound-orders/:id`
  - `/api/v1/outbound-orders/:id`
- **建议方案**: 
  - 为 ID 字段添加数据库索引
  - 优化查询语句，减少不必要的关联查询
  - 考虑缓存策略

### API-001: 批量操作接口方法不匹配
- **严重程度**: 🟢 轻微
- **状态**: ⏳ 待处理
- **描述**: 批量导入、批量标签等接口只支持 POST 方法，但测试脚本使用 GET 方法调用，返回 405 错误
- **影响接口**:
  - `/api/customers/batch-import`
  - `/api/customers/batch-tag`
  - `/api/v1/products/convert-from-research`
- **建议方案**: 更新测试脚本，对这些接口使用正确的 HTTP 方法

### API-002: 报表接口缺少必要参数
- **严重程度**: 🟢 轻微  
- **状态**: ⏳ 待处理
- **描述**: 报表类接口需要日期范围等查询参数，无参数调用返回 400 错误
- **影响接口**:
  - `/api/v1/reports/sales`
  - `/api/v1/reports/profit`
  - `/api/v1/reports/purchase`
  - `/api/v1/reports/cashflow`
- **建议方案**: 在测试中提供默认的查询参数，如 `?startDate=2026-04-01&endDate=2026-04-09`
