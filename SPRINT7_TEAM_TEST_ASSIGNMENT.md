## 📝 各角色测试任务

### 1️⃣ 需求工程师 - 需求验证

**任务目标：** 验证 v0.7.0 是否达到 Sprint 7 的原始需求目标

**检查清单：**

```
□ 1. 认证方案需求验证
  □ 1.1 是否解决了 NextAuth v5 与 Next.js 16 的兼容性问题？
  □ 1.2 是否实现了 JWT 简化认证方案？
  □ 1.3 是否支持 7 天会话有效期？
  
□ 2. 功能需求验证
  □ 2.1 用户登录功能是否正常？
  □ 2.2 用户登出功能是否正常？
  □ 2.3 认证状态检查是否正常？
  □ 2.4 权限控制是否正常？
  
□ 3. 安全需求验证
  □ 3.1 密码是否加密存储（bcrypt）？
  □ 3.2 JWT 是否使用 HS256 签名？
  □ 3.3 Cookie 是否设置 HttpOnly + Secure？
  □ 3.4 是否实现登录速率限制（5 次/15 分钟）？
  
□ 4. 兼容性需求验证
  □ 4.1 是否与现有中间件兼容？
  □ 4.2 是否与现有 API 兼容？
  □ 4.3 是否与前端页面向兼容？
```

**交付物：** `docs/Sprint7-Requirement-Verification.md`

**预计用时：** 30 分钟

**任务目标：** 评估认证重构的技术架构和代码质量

**检查清单：**

```
□ 1. 架构设计评审
  □ 1.1 认证模块结构是否清晰？
  □ 1.2 职责分离是否合理（auth-simple.ts、中间件、API）？
  □ 1.3 是否遵循单一职责原则？
  
□ 2. 代码质量评审
  □ 2.1 代码注释是否完整？
  □ 2.2 错误处理是否完善？
  □ 2.3 日志输出是否合理？
  
□ 3. 安全性评审

□ 4. 性能评估
  □ 4.1 认证 API 响应时间 < 500ms？

  □ 4.3 数据库查询是否优化？
  
□ 5. 可维护性评估
  □ 5.1 代码是否易于理解和修改？
  □ 5.2 是否有完整的文档？
```

**交付物：** `docs/Sprint7-Architecture-Review.md`

### 3️⃣ 前端工程师 - 前端功能测试

**任务目标：** 验证前端页面认证功能和用户体验

**检查清单：**

```
□ 1. 登录页面测试
  □ 1.1 登录页面是否正常加载？
  □ 1.2 表单验证是否正常（空账号/空密码）？
  □ 1.3 登录成功是否跳转首页？
  □ 1.4 登录失败是否显示错误提示？
  
□ 2. 认证状态测试
  □ 2.1 登录后刷新页面是否保持登录？
  □ 2.2 未登录访问受保护页面是否跳转登录？
  □ 2.3 用户信息显示是否正确？
  
□ 3. 登出功能测试
  □ 3.1 点击退出登录是否成功？
  □ 3.2 登出后是否跳转登录页？
  □ 3.3 登出后返回是否无法访问受保护页面？
  
□ 4. 权限控制测试
  □ 4.1 不同角色是否看到不同菜单？
  □ 4.2 无权限访问是否提示？
  
□ 5. 浏览器兼容性测试
  □ 5.1 Chrome 浏览器是否正常？
  □ 5.2 Safari 浏览器是否正常？
  □ 5.3 移动端浏览器是否正常？
  
□ 6. UI/UX 测试
  □ 6.1 登录表单样式是否正常？
  □ 6.2 错误提示是否友好？
  □ 6.3 加载状态是否显示？
```

**交付物：** `docs/Sprint7-Frontend-Test-Report.md`

**预计用时：** 45 分钟

### 4️⃣ 后端工程师 - API 功能测试

**任务目标：** 验证认证 API 和后端逻辑

**检查清单：**

```
□ 1. 认证 API 测试
  □ 1.1 POST /api/auth/login - 登录成功
  □ 1.2 POST /api/auth/login - 密码错误
  □ 1.3 POST /api/auth/login - 用户不存在
  □ 1.4 POST /api/auth/login - 参数验证
  
□ 2. 会话 API 测试
  □ 2.1 GET /api/auth/me - 已认证
  □ 2.2 GET /api/auth/me - 未认证
  
□ 3. 登出 API 测试
  □ 3.1 POST /api/auth/logout - 登出成功
  □ 3.2 POST /api/auth/logout - Cookie 清除
  
□ 4. 速率限制测试
  □ 4.1 连续 5 次错误登录 - 返回 401
  □ 4.2 第 6 次错误登录 - 返回 429
  □ 4.3 15 分钟后重试 - 恢复正常
  
□ 5. 数据库验证
  □ 5.1 用户密码是否加密存储？
  □ 5.2 用户数据是否完整？
  
□ 6. 中间件测试
  □ 6.1 requireAuth 中间件是否正常工作？
  □ 6.2 权限验证是否正确？
```
