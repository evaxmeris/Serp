## ⚠️ 发现的问题

### 1. API 错误处理不一致

**问题**: PUT 和 DELETE 操作在记录不存在时返回 500 而不是 404

**位置**: `src/app/api/products/[id]/route.ts`

**当前行为**:
```typescript
// Prisma 抛出 P2025 错误，被捕获后返回 500
catch (error) {
  return NextResponse.json(
    { error: 'Failed to update/delete product' },
    { status: 500 }
  );
}
```

**建议修复**:
```typescript
catch (error) {
  if (error.code === 'P2025') {
    return NextResponse.json(
      { error: 'Product not found' },
      { status: 404 }
    );
  }
  return NextResponse.json(
    { error: 'Failed to update/delete product' },
    { status: 500 }
  );
}
```

**优先级**: 中

### 2. 状态筛选枚举值问题

**问题**: 产品模型的 `status` 字段是 Prisma 枚举类型，但 API 直接透传字符串值

**影响**: 使用无效状态值（如 "active"）会导致 500 错误

**建议**: 
- API 层添加状态值验证
- 或返回空列表而不是错误

**优先级**: 低
