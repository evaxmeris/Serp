## 🔧 Bug 修复

### 严重问题修复

#### BUG-AUTH-001: JWT 密钥不一致导致认证失败

**问题：** `auth-simple.ts` 和 `auth-api.ts` 两个模块使用静态常量保存密钥，由于 Next.js 模块缓存机制，导致两个模块得到的密钥不一致，签名验证永远失败，所有 API 返回 401。

**修复：** 改为每次使用时动态从环境变量获取密钥，保证签名和验证使用同一个密钥。

**影响范围：** 所有需要认证的 API 接口。

**状态：** ✅ 已修复

---

#### API-001: 供应商 API 中间件不兼容

**问题：** 供应商模块原使用旧版 `requireAuth` 中间件，只检查 Bearer Token，与新版 cookie 认证不兼容。

**修复：** 更新供应商模块所有 API 使用新版 `auth-simple.ts` 中间件。

**影响范围：** 供应商管理相关 API。

**状态：** ✅ 已修复

---

#### 其他修复

- **API-002:** 批量操作接口方法不匹配 - 文档已更新，明确要求使用 POST 方法
- **UI-001:** 缺少错误边界处理 - 已添加 ErrorBoundary 组件（v0.8.0）
