## ⚠️ 已知问题

### 问题 1：供应商 API 中间件不兼容

**现象：** API 直接调用返回 401，但前端页面可用

**原因：** 供应商 API 使用旧版 `requireAuth` 中间件（检查 Bearer Token），与新版 cookie 认证不兼容

**影响：** 仅影响 API 直接调用，前端页面通过 `auth-simple.ts` 认证，不受影响

**解决方案：** 
- 短期：前端页面正常使用
- 长期：更新供应商 API 中间件使用 `auth-simple.ts`

**优先级：** 🟡 中（不影响发布）
